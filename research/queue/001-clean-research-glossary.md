# 정리된 연구 용어 사전 설계

> 상태: `QUEUED`
>
> 현재 PR #31의 이론 내용 수정 범위에는 포함하지 않는다.

## 문제

현재 `chapters/`와 연구 문서에는 같은 이름 아래 여러 책임이 섞였거나, 같은 개념이 여러 이름으로 흩어진 경우가 있다.

향후 용어 사전을 만들 때 이 엉킨 사용을 그대로 수집·전시해서는 안 된다.

사전의 목적은 역사적 혼란을 모두 수용하는 것이 아니라, 연구에서 줄을 충분히 풀어낸 뒤 현재 사용할 수 있게 정리된 용어를 모아 두는 것이다.

## 향후 처리할 일

- `chapters/`와 현재 연구 항목의 내용을 다시 읽는다.
- 한 용어 아래 섞인 서로 다른 책임을 분리한다.
- 여러 이름으로 흩어진 동일한 개념은 합칠 수 있는지 판정한다.
- 폐기할 역사적 이름과 현재 사용할 이름을 구분한다.
- 각 연구 항목에서 사용하는 정리된 지역 용어와 연구 전체가 공유할 용어의 관계를 정한다.
- 그 분석 뒤에만 지역 사전과 전체 사전의 실제 구조를 설계한다.

## 이 작업에서 하지 않을 것

- 과거의 모든 정의·기호·판본을 사전 본문에 보존하지 않는다.
- 역사적 용어 충돌을 사전의 중심 구조로 만들지 않는다.
- 사전을 계보 레지스트리나 실행 Registry로 만들지 않는다.
- 현재 단계에서 파일 형식, 항목 schema, 디렉터리 구조를 미리 고정하지 않는다.
- 정리되지 않은 용어를 단지 자주 등장했다는 이유로 전역 용어로 승격하지 않는다.

## 처리 시점

PR #31에서 진행 중인 연구 문서의 개념 경계 수정이 끝난 뒤 별도 문서 작업으로 처리한다.

이 항목을 처리할 때는 당시의 `chapters/`, 연구 thread, crossing 문서를 다시 기준으로 삼는다.

## 완료 조건

- 사전에 들어갈 대상이 역사적 용어 목록이 아니라 정리된 현재 개념임이 명확하다.
- 각 용어가 무엇을 책임지고 무엇과 구분되는지 설명할 수 있다.
- 지역 용어와 전역 공유 용어의 관계가 실제 연구 내용에서 도출된다.
- 역사적 혼란과 변천은 `chapters/` 및 별도 계보 문서에 남고, 사전 본문에는 정리 결과만 들어간다.
- 사전 구조는 위 분석 결과를 바탕으로 정하며 사전에 미리 결정하지 않는다.
