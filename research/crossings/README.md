# Crossings

`crossings/`는 둘 이상의 연구 흐름에 걸쳐 있지만 아직 어느 흐름의 소유인지, 독립된 문제 공간인지 판정할 수 없는 연구를 둔다.

교차 문제를 즉시 새 thread로 승격하지 않는 이유는 다음과 같다.

- 여러 흐름의 단순 결합일 수 있다.
- 한 흐름의 하위 문제인데 다른 흐름의 언어를 빌리고 있을 수 있다.
- 서로 다른 현상이 하나의 이름 아래 섞여 있을 수 있다.
- 독립 개념처럼 보이지만 별도 예측을 만들지 못할 수 있다.

---

## 1. crossing 문서가 답할 질문

```text
어떤 연구 흐름들이 만나는가?
각 흐름에서 이 문제를 어떻게 다르게 보는가?
공통된 현상은 무엇인가?
독립적인 객체나 메커니즘이 정말 필요한가?
한 흐름으로 환원하면 무엇을 잃는가?
독립 thread로 만들면 무엇을 새로 예측할 수 있는가?
```

---

## 2. 현재 대표 교차 문제

### 재저자화

과거 발생의 보존, 현재 의미의 변경, 책임과 자기 소유, 자기 피드백, 타인의 권리가 동시에 얽힌다.
현재 독립적인 단일 메커니즘이라고 확정하지 않는다.

### 자원과 시간

피로와 시간 압력이 후보 생성, 선택, 행동 실행, 상태 지속, 회복 가운데 어디에 작용하는지 불명확하다.

### Narrative와 사회성

Narrative가 개인 내부의 역사 구조인지, 타자와의 반복 관계인지, 공유되지 않는 복수 관계 모델인지 아직 구분되지 않았다.

---

## 3. crossing의 가능한 이후 경로

### 독립 thread가 된다

서로 다른 흐름에 환원되지 않는 질문·예측·실패 조건이 확인된 경우.

### 기존 thread에 흡수된다

한 흐름의 언어와 메커니즘만으로 충분히 설명되는 경우.

### 관계 문제로 남는다

새 객체가 아니라 두 흐름 사이의 연결 규칙이라는 점이 드러난 경우.

### 여러 문제로 해체된다

하나의 이름 아래 서로 다른 현상이 섞였음이 드러난 경우.

### 폐기된다

별도 개념이 불필요하거나 실제 수행 차이를 만들지 못하는 경우.

---

## 4. crossing을 사용할 때의 주의

- 귀속을 결정하지 못했다는 이유로 모호한 종합문을 쓰지 않는다.
- 여러 흐름의 용어를 연결했다고 새 이론을 얻었다고 보지 않는다.
- 교차 문제를 상위 통합 구조로 취급하지 않는다.
- 독립성을 주장하려면 독립적인 판별 차이가 필요하다.
- crossing이 오래 유지되는 것 자체도 현재 분류가 잘못되었다는 신호일 수 있다.

`crossings/`는 구조의 빈칸을 감추는 곳이 아니라, 아직 경계를 모른다는 사실을 명시적으로 보존하는 곳이다.
