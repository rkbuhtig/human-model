# 초기 이론 내용 지도

> **지위:** 현재 작업 지도
>
> 이 문서는 각 연구 흐름 안에 들어간 초기 이론 내용을 찾기 위한 색인이다. 아래 문서 목록은 완성된 이론 구조, 영구 축, 상위 조립 순서를 뜻하지 않는다. 각 문서는 현재의 넓은 문제 묶음 안에서 무엇을 먼저 구분할지 적은 작업물이다.

## 6. Crossings

교차 문제는 독립된 이론 축으로 승인된 것이 아니다. 여러 흐름에서 같은 이름과 대상 경계가 충돌하지만 어느 한쪽에 조기 귀속할 수 없을 때 임시로 둔 문제 공간이다.

Crossing은 부모 thread의 working model을 다시 합치거나 상위 Assembly를 만드는 곳이 아니다.

### 재저자화

- [`crossings/reauthoring/README.md`](../crossings/reauthoring/README.md)

중심 충돌:

```text
과거 발생 보존
vs
현재 의미·저자성·지지·책임·자기정의·미래 정책의 재판정
```

경쟁 설명:

```text
의미 재해석
기억 재구성
책임 정산
Narrative 편집
정책 학습
사회적 협상
```

`reauthoring`은 이 설명들을 하나로 묶은 확정 메커니즘이 아니다.

### 자원과 시간

- [`crossings/resource-and-time/README.md`](../crossings/resource-and-time/README.md)

현재 분리하는 조건:

```text
capacity
= 현재 탐색·통제·수행에 동원할 수 있는 능력

clock
= 과정에 주어진 시간과 사건 사이 실제 경과 시간

stake / hazard
= 행동과 표현에 걸린 예상 손실과 위험

option constraint
= 현재 선택이 미래 경로를 닫거나 비싸게 만드는 정도
```

이 네 종류는 확정 ontology가 아니라 `resource`라는 한 말 아래 다른 종류의 제약을 섞지 않기 위한 비교 언어다.

### Narrative와 사회성

- [`crossings/narrative-and-sociality/README.md`](../crossings/narrative-and-sociality/README.md)

중심 residence 후보:

```text
개인 내부 자기서사와 관계 모델
개인별 관계 기록
실제 상호작용 역사
상호 인정된 기록
명시적 합의
결합 동역학
언어적 관계 설명
관찰자 요약
```

이 crossing은 하나의 사회적 Narrative 이론을 만드는 대신, `Narrative`, `관계`, `기록`, `공유`라는 이름이 서로 다른 대상을 오가는 문제를 다룬다.

## 7. 문장 종류의 경계

현재 문서들은 다음 종류의 문장을 함께 다룰 수 있지만 서로를 같은 근거로 사용하지 않는다.

```text
서술 가설
= 인간의 상태와 수행이 실제로 어떻게 변할 수 있는가

사실 경계
= 무엇이 실제로 발생했고 어떤 증거로 알 수 있는가

규범적 경계
= 책임, 권리, 동의, 관계 조건이 어떻게 판정되는가

관측 경계
= 어떤 표면과 후속 수행으로 내부 변화를 구분할 수 있는가

모델링 언어
= 경쟁 설명과 구현 가능성을 비교하기 위한 표현
```

규범적으로 바람직한 구분이 인간 내부의 독립 구조라는 증거는 아니며, 엔진과 LLM에서 유용한 언어가 인간 메커니즘이라는 증거도 아니다.

## 8. 현재 문서 묶음이 의도적으로 하지 않는 것

- Episode, Narrative, Self, Persona를 독립 정본 디렉터리로 만들지 않는다.
- 현재 다섯 thread를 인간의 불변 축으로 승인하지 않는다.
- 문서 사이의 연결을 완성된 상위 이론으로 조립하지 않는다.
- crossing을 독립 상위 메커니즘으로 승인하지 않는다.
- 경쟁 설명 가운데 하나를 자동 채택하지 않는다.
- 프롬프트 구현 형태를 먼저 고정하지 않는다.
- 문헌 인접성과 공학적 유용성을 인간 경험적 지지로 승격하지 않는다.
- 모든 문서에 고정 claim schema를 강제하지 않는다.
- 이 지도에서 용어 사전이나 전역 ontology를 만들지 않는다.

## 9. 이 내용을 사용하는 방법

현재 문서는 다음 순서로 읽을 수 있다.

```text
thread README
→ current working model
→ challenges / competing explanations
→ 필요한 crossing 문서
→ 실제 프롬프트 모델이나 상호작용에서 차이를 확인
→ 발견된 충돌을 원래 thread 또는 별도 이슈에 기록
→ 경계가 바뀌면 transitions/에 구조 변경 이유를 남김
```

문서의 목적은 하나의 매끄러운 설명을 제공하는 것이 아니다.

> 각 문제 묶음이 무엇을 설명하려 하는지, 어떤 구분을 임시로 사용하며, 어떤 결과 앞에서 갈라지거나 합쳐지거나 폐기되어야 하는지를 실제 연구 가능한 형태로 만드는 것.
