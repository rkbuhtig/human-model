## 7. 현재 문서 묶음이 의도적으로 하지 않는 것

- Episode, Narrative, Self, Persona를 독립 정본 디렉터리로 만들지 않는다.
- 현재 다섯 thread를 인간의 불변 축으로 승인하지 않는다.
- 문서 사이의 연결을 완성된 상위 이론으로 조립하지 않는다.
- 경쟁 설명 가운데 하나를 자동 채택하지 않는다.
- 프롬프트 구현 형태를 먼저 고정하지 않는다.
- 문헌 인접성을 인간 경험적 지지로 승격하지 않는다.
- 모든 문서에 고정 claim schema를 강제하지 않는다.

---

## 8. 이 내용을 사용하는 방법

현재 문서는 다음 순서로 읽을 수 있다.

```text
thread README
→ current working model
→ challenges / competing explanations
→ crossing 문서
→ 필요한 경우 질문을 실제 프롬프트 모델이나 상호작용 평가로 가져감
→ 발견된 충돌을 다시 thread에 기록
→ 경계가 바뀌면 transitions/에 구조 변경 이유를 남김
```

문서의 목적은 하나의 매끄러운 설명을 제공하는 것이 아니다.

> 각 문제 묶음이 무엇을 설명하려 하는지, 어떤 구분을 임시로 사용하며, 어떤 결과 앞에서 갈라지거나 합쳐지거나 폐기되어야 하는지를 실제 연구 가능한 형태로 만드는 것.
