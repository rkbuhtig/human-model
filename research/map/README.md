# Current Research Map

`map/`은 현재 연구 지형을 보여주는 임시 지도다.
이 지도는 완성된 이론 구조나 최종 dependency graph가 아니다.

디렉터리 구조보다 빠르게 바뀔 수 있으며, 현재 어떤 문제를 어디에서 보고 있는지와 어디에서 경계가 흔들리는지를 기록한다.

---

## 1. 현재 활성 연구 흐름

| 연구 흐름 | 현재 중심 질문 | 주요 중첩 |
|---|---|---|
| `events-and-change` | 무엇이 발생하고 어떤 의미에서 변화가 확정되는가 | 지속, 자기 피드백, 재저자화 |
| `persistence-and-history` | 무엇이 남고 과거가 현재와 미래를 어떻게 기울이는가 | 사건, 피드백, 자기·타자 |
| `possibility-and-action` | 여러 가능성이 어떻게 생기고 실제 행동으로 이어지는가 | 자원, 표현, 피드백 |
| `self-feedback` | 행동과 결과가 자신을 어떻게 다시 바꾸는가 | 지속, 정산, 타자 반응 |
| `self-other-surface` | 내부·타자 모델·관계가 표현과 행동을 어떻게 형상화하는가 | 가능성, 피드백, 역사 |

이 표의 행은 불변 축이 아니다.
현재 질문을 추적하기 위한 작업용 묶음이다.

---

## 2. 현재 교차 문제

### 재저자화

현재 다음 흐름을 동시에 건드린다.

```text
events-and-change
- 과거 발생과 현재 의미는 어떻게 다른가
- 책임과 자기 소유는 어떤 의미에서 확정되는가

persistence-and-history
- 과거 사건은 어떤 형태로 다시 접근되는가
- 새로운 정보가 오래된 잔여를 어떻게 바꾸는가

self-feedback
- 후회와 결과가 과거 행동에 대한 현재 관계를 어떻게 바꾸는가

self-other-surface
- 나의 재해석과 타인의 권리·기억·평가는 어떻게 분리되는가
```

현재 독립 thread로 승격하지 않는다.

### 자원과 시간

피로·시간·주의 제한이 다음 중 무엇인지 아직 불명확하다.

```text
가능성 생성 폭의 조건
선택 비용
행동 실행 능력
회복 속도
장기 지속의 원인
```

현재는 `possibility-and-action`을 중심으로 보되 `persistence-and-history`, `self-feedback`과의 경계를 열어둔다.

### Narrative와 사회성

Narrative가 개인 내부의 느린 구조인지, 관계 속 반복 경로인지, 타자 모델의 압축인지 아직 구분되지 않았다.
현재 `persistence-and-history`와 `self-other-surface` 사이의 교차 문제로 본다.

## 3. 현재 해결되지 않은 경계

### 발생과 정산

```text
무슨 일이 일어났는가
vs
그 일을 내 행동으로 인수하는가
vs
그 행동을 지지하는가
vs
책임을 수용하는가
```

이들이 하나의 연속 과정인지 서로 다른 정산인지 아직 닫히지 않았다.

### 권한과 인과력

무언가가 미래 선택을 바꿀 수 있다는 사실과, 사실·책임·타인의 권리를 확정할 수 있다는 것은 어떻게 다른가.
이 차이가 독립 원리인지 여러 경계의 공통 표현인지 열려 있다.

### 기억과 잔여

```text
과거 내용을 다시 말할 수 있음
다음 주의와 기대가 바뀜
같은 상황의 행동분포가 바뀜
전이법 자체가 바뀜
```

이 네 효과를 모두 기억이라고 불러야 하는지 불명확하다.

### Episode

```text
실제로 걸은 경로
경로가 남긴 상태
나중에 접근하는 기록
하나의 의미 단위
```

현재 같은 이름 아래 복수 기능이 섞여 있을 가능성이 높다.

### Narrative

```text
활성화된 자기서사 내용
느린 내부 상태
전이 경사의 변화
여러 경로의 파생 graph
관계 속 공유 또는 충돌하는 해석
```

현재 독립 객체로 확정하지 않는다.

### Self와 Persona

반복되는 수행이 다음 중 무엇인지 열려 있다.

```text
느린 상태
전이 편향
자원 사용 습관
자기 저자화 경향
타자 모델링 방식
관찰자가 만든 요약
```

현재 별도 연구 흐름을 만들지 않는다.

## 4. 현재 구조 변경 후보

현재 당장 수행하지 않지만 관찰할 후보들이다.

### `events-and-change` 분리 가능성

발생·정산·권한이 서로 다른 예측과 실패 조건을 가진다면 분리할 수 있다.

### `self-other-surface` 분리 가능성

타자 모델의 갱신과 상황적 표현이 독립적으로 움직인다면 `social-modeling`과 `situated-expression`으로 나눌 수 있다.

### `resource-pressure` 독립 가능성

자원 변화가 가능성·행동·회복에 걸쳐 공통된 독립 구조를 보인다면 별도 흐름이 될 수 있다.

### `self-feedback` 개명 가능성

핵심이 자기평가보다 결과를 통한 전이법 변화라면 `outcome-driven-adaptation`이 더 정확할 수 있다.

이 후보들은 계획이 아니라 관찰 포인트다.

---

## 5. 지도 수정 원칙

- 새로운 문제를 발견하면 먼저 인접 흐름과 교차 여부를 기록한다.
- 현재 구획에 잘 맞지 않는다는 이유로 즉시 새 thread를 만들지 않는다.
- 경계 변화가 반복되거나 독립 예측이 드러날 때 구조 변경을 검토한다.
- 구조 변경 이유는 `transitions/`에 남긴다.
- 폐기된 구획의 과거 역할도 보존한다.
