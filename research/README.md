# Research

`research/`는 인간 이론의 완성된 구조를 미리 선언하는 곳이 아니다.
이곳은 반복해서 나타난 문제를 현재의 연구 단위로 묶어 탐구하고, 그 묶음 자체가 적절한지 계속 다시 판정하는 작업 공간이다.

이 디렉터리에서 가장 중요한 전제는 다음과 같다.

> **현재의 연구 구획은 이론의 영구 축이 아니다.**
>
> 각 구획은 지금 함께 탐구할 가치가 있다고 판단한 임시 문제 흐름이며,
> 연구 결과에 따라 세분화·병합·개명·흡수·이동·중단·해체될 수 있다.

따라서 현재 보이는 디렉터리 구조를 인간의 최종 존재론, 독립된 심리 모듈, 확정된 불변량 목록, 완성된 이론의 목차로 읽어서는 안 된다.
구조 자체도 연구 대상이다.

---

## 1. 왜 고정된 축을 만들지 않는가

현재 연구에는 다음과 같은 반복 문제가 보인다.

- 무엇이 실제로 발생했고 무엇이 현재 해석인가
- 생각과 행동 가능성은 어떻게 열리고 제한되는가
- 어떤 변화가 얼마나 오래 남는가
- 자신의 행동과 결과가 어떻게 자신에게 돌아오는가
- 타인을 어떻게 추정하고 그 추정이 표현을 어떻게 바꾸는가
- 과거 행동의 의미와 책임이 나중에 어떻게 다시 판정되는가
- 시간·피로·주의·관계 위험이 선택을 어떻게 바꾸는가

이 문제들은 분명 중요하지만, 아직 다음이 확정되지 않았다.

```text
서로 정말 독립적인가?
한 문제가 다른 문제의 하위 현상은 아닌가?
하나의 이름 아래 여러 현상이 섞여 있지 않은가?
현재 경계가 프롬프트 엔진의 구현 습관을 인간 구조로 오해한 결과는 아닌가?
새로운 평가가 나오면 질문 자체가 바뀌지 않는가?
```

지금 `authority`, `temporality`, `scarcity`, `narrative` 같은 이름을 영구 축으로 고정하면,
이후의 연구가 그 축을 검증하는 대신 그 축을 보존하는 방향으로 기울 수 있다.

예를 들어 다음 가능성은 모두 열려 있다.

```text
권한 문제와 정산 문제가 실제로는 하나의 문제일 수 있다.
권한 문제 안에 서로 다른 두 종류의 경계가 섞여 있을 수 있다.
희소성은 독립 축이 아니라 선택 과정의 조건일 수 있다.
Narrative는 독립 객체가 아니라 시간적 지속 연구에서 나온 파생 설명일 수 있다.
재저자화는 하나의 메커니즘이 아니라 여러 문제 흐름의 교차점일 수 있다.
```

따라서 이 저장소는 현재의 연구 구획을 **축(axis)** 대신 **연구 흐름(thread)** 또는 **문제 공간(problem space)** 으로 다룬다.

---

## 2. 연구 흐름이란 무엇인가

연구 흐름은 하나의 확정된 이론 부품이 아니다.
현재 여러 질문과 자료를 같은 자리에서 비교하는 편이 생산적이라고 판단한 임시 연구 단위다.

연구 흐름은 다음을 포함할 수 있다.

- 아직 정확히 분리되지 않은 여러 현상
- 서로 경쟁하는 설명
- 과거 자료에서 반복된 구조
- 현재 대화에서 새로 얻은 통찰
- 프롬프트 모델에서 발견된 실패
- 다른 흐름과의 경계 문제
- 현재 이름으로는 잘 포착되지 않는 잔여

연구 흐름의 목표는 자기 정당화가 아니다.
오히려 다음을 알아내는 것이다.

```text
이 문제 묶음은 실제로 하나인가?
어디에서 갈라져야 하는가?
다른 흐름과 무엇을 공유하는가?
어떤 설명은 폐기되어야 하는가?
현재 이름이 문제의 본질을 가리고 있지 않은가?
```

하나의 연구 흐름이 해체되거나 다른 흐름에 흡수되는 것은 실패가 아니다.
더 나은 구분을 발견했다는 연구 결과일 수 있다.

---

## 3. 허용되는 구조 변화

모든 연구 흐름은 다음 변화를 허용한다.

### 세분화

하나의 흐름 안에서 서로 다른 원인·시간척도·객체·예측이 드러나면 둘 이상으로 나눈다.

```text
events-and-change
→ occurrence
→ settlement
→ authority
```

위와 같은 분리는 미리 예정하지 않는다.
실제 연구에서 세 문제가 서로 다른 실패 조건과 예측을 갖는다는 점이 확인될 때 수행한다.

### 병합

서로 다른 흐름이 같은 현상을 같은 구조로 설명하고 독립적인 예측을 내지 못한다면 하나로 합칠 수 있다.

```text
resource-pressure
+ possibility-and-action
→ constrained-action
```

### 개명

현재 이름이 연구 질문을 잘못 유도하거나, 연구 중 중심 문제가 달라졌다면 이름을 바꾼다.

```text
self-feedback
→ outcome-driven-adaptation
```

개명은 단순 표현 수정이 아니라 연구 관점의 이동일 수 있으므로 `transitions/`에 이유를 남긴다.

### 일부 이동

흐름 전체를 바꾸지 않고 특정 질문·자료·발견만 다른 흐름으로 옮길 수 있다.

### 흡수

독립 흐름으로 유지할 이유가 사라지고 다른 흐름의 하위 문제로 설명된다면 흡수한다.

### 잠정 중단

중요하지만 현재 자료나 평가 방법이 부족한 경우 `dormant/`로 이동한다.
이는 폐기가 아니다.

### 해체

문제 묶음 자체가 잘못되었다면 각 자료를 다른 곳으로 옮기고 흐름을 닫는다.

### 폐기

설명이나 연구 방향이 반례, 중복, 개념 오류, 실제 수행 실패로 약화되었다면 `discarded/`에 이유와 잔여 가치를 남긴다.

## 4. 현재 디렉터리 구조

```text
research/
├─ README.md
├─ map/
│  └─ README.md
├─ threads/
│  ├─ README.md
│  ├─ events-and-change/
│  │  └─ README.md
│  ├─ persistence-and-history/
│  │  └─ README.md
│  ├─ possibility-and-action/
│  │  └─ README.md
│  ├─ self-feedback/
│  │  └─ README.md
│  └─ self-other-surface/
│     └─ README.md
├─ crossings/
│  └─ README.md
├─ transitions/
│  └─ README.md
├─ dormant/
│  └─ README.md
└─ discarded/
   └─ README.md
```

이 구조에서 아직 만들지 않는 것은 다음과 같다.

```text
assemblies/
완성된 human-dynamics/
Episode/Narrative/Self/Persona 전용 정본 디렉터리
고정된 invariant registry
최종 ontology
의존성 그래프를 정답처럼 동결한 manifest
```

이들은 현재 연구가 실제 경계를 발견하기 전에 미래 이론의 모양을 선점할 수 있기 때문이다.

## 5. `map/` — 현재 지형을 보여주는 임시 지도

`map/`은 이론의 정답 구조가 아니다.
현재 어떤 연구 흐름이 열려 있고, 서로 어디에서 중첩되며, 어떤 경계가 해결되지 않았는지를 보여준다.

여기서는 다음을 관리한다.

- 현재 활성 연구 흐름
- 각 흐름의 중심 질문
- 흐름 사이의 중첩
- 어느 곳에도 아직 귀속되지 않는 문제
- 분리나 병합을 검토 중인 경계
- 최근 구조 변화

지도는 연구 결과에 따라 자주 바뀔 수 있다.
디렉터리 구조와 지도가 일치하지 않는 순간이 생길 수도 있다.
그 경우 지도를 현재 판단의 우선 기록으로 사용하고, 구조 변경은 충분한 이유가 생긴 뒤 수행한다.

## 6. `threads/` — 현재 독립적으로 밀어보는 문제 공간

현재 다섯 개 흐름을 연다.

### `events-and-change/`

무엇이 실제로 발생했는지, 해석과 발생을 어떻게 구분할지, 변화가 어떤 의미에서 확정되는지, 무엇이 무엇을 바꿀 수 있는지를 함께 조사한다.

현재는 occurrence·settlement·authority를 별도 축으로 확정하지 않는다.
연구가 실제 차이를 보일 때만 분리한다.

### `persistence-and-history/`

무엇이 시간 속에서 남고, 과거가 어떤 형태로 현재와 미래에 작동하며, Episode·Narrative·기억·잔여·재해석 같은 개념이 실제로 어떻게 갈리는지 조사한다.

### `possibility-and-action/`

여러 생각과 행동 가능성이 어떻게 생기고, 상황과 자원 압력이 이를 어떻게 제한하며, 어떻게 실제 행동이나 보류로 이어지는지 조사한다.

현재는 분포·선택·충동·습관·심의·희소성을 따로 고정하지 않는다.

### `self-feedback/`

자신의 행동, 세계 결과, 타인의 반응이 어떻게 자신에게 돌아오며, 후회·수치·만족·합리화·책임 수용·회복·학습이 무엇을 실제로 바꾸는지 조사한다.

### `self-other-surface/`

내부 상태와 표현이 왜 같지 않은지, 타인을 어떻게 추정하는지, 타인이 보는 자신을 어떻게 상상하는지, 관계·역할·공개성·체면이 표현과 행동을 어떻게 바꾸는지 조사한다.

이 다섯 개는 최종 축이 아니다.
현재 연구를 시작하기에 충분히 넓고, 서로의 경계를 조사할 수 있도록 만든 첫 문제 묶음이다.

---

## 7. `crossings/` — 아직 귀속할 수 없는 교차 문제

여러 연구 흐름을 동시에 건드리지만, 독립 흐름인지 한쪽의 하위 문제인지 아직 판정할 수 없는 문제를 둔다.

예를 들어 재저자화는 다음을 동시에 건드린다.

- 과거 발생과 현재 의미의 구분
- 시간에 따른 재접근
- 행동 결과와 자기 피드백
- 책임과 자기 소유의 정산
- 타인의 권리와 관계적 결과

이때 곧바로 `threads/reauthoring/`을 만들지 않는다.
먼저 `crossings/`에서 어떤 흐름들이 실제로 필요한지, 독립적인 실패 조건과 예측이 있는지 조사한다.

교차 문제는 이후 다음 중 하나가 될 수 있다.

```text
독립 흐름으로 이동
기존 흐름 하나에 흡수
여러 흐름을 연결하는 관계 문제로 유지
잘못 묶인 문제로 해체
```

---

## 8. `transitions/` — 구조 변화 자체를 연구 결과로 기록

디렉터리의 분리·병합·개명·이동·흡수·해체는 단순 파일 정리가 아니다.
연구가 문제의 본질을 다르게 이해하게 되었다는 기록일 수 있다.

따라서 중요한 구조 변화는 `transitions/`에 남긴다.

기록에는 다음을 포함한다.

```text
이전 구조
이전 구조가 유용했던 이유
어떤 문제가 드러났는가
새 구조
어떤 자료가 어디로 이동했는가
아직 해결되지 않은 부분
이 변화가 이론적으로 뜻하는 것
```

Git 이력은 파일의 변화를 보존하지만, 왜 경계를 다시 그었는지는 설명하지 못한다.
`transitions/`는 그 이유를 연구 기억으로 보존한다.

---

## 9. `dormant/` — 지금 밀지 않는 연구

다음과 같은 연구를 둔다.

- 중요하지만 지금 필요한 자료가 없음
- 현재 모델이나 평가로 구분할 수 없음
- 다른 흐름이 먼저 정리되어야 함
- 연구 우선순위상 잠시 중단됨
- 개념은 살아 있지만 현재 이름이나 경계가 불명확함

잠정 중단은 폐기와 다르다.
현재 mainline에서 사용하지 않지만 다시 열 조건을 남긴다.

---

## 10. `discarded/` — 실패를 삭제하지 않는 연구 기억

폐기된 설명과 연구 방향을 보존한다.

폐기 기록은 다음을 남겨야 한다.

- 무엇을 설명하려 했는가
- 왜 유망하다고 보았는가
- 어떤 자료와 모델에 연결되었는가
- 어디에서 실패했는가
- 단순 구현 실패인지 이론 실패인지
- 어떤 설명이 이를 대체하는가
- 여전히 회수할 수 있는 부분은 무엇인가
- 어떤 조건이라면 다시 검토할 수 있는가

폐기 디렉터리는 현재 이론의 비공식 저장소가 아니다.
다시 회수할 때에는 어디에서 왜 회수했는지 밝힌다.

---

## 11. 하나의 연구 흐름 안에서 하는 일

각 흐름의 구체적인 하위 구조는 필요에 따라 달라질 수 있다.
현재 권장하는 최소 작업은 다음과 같다.

### 질문을 적는다

무엇을 아직 구분하지 못했는지 적는다.
좋은 질문은 현재 디렉터리 경계를 의심할 수 있어야 한다.

### 자료를 모은다

- 기존 `chapters/`, `notes/`, `checkpoints/`
- 페르소나 엔진 자료
- 현재 대화의 통찰
- 모델 평가 기록
- 인접 문헌과 공개 데이터

자료가 있다는 사실은 이론적 지지와 다르다.
자료는 문제를 다시 보게 하는 입력이다.

### 여러 설명을 병렬로 둔다

하나의 개념 이름으로 빠르게 닫지 않는다.

예:

```text
Narrative는 느린 상태다.
Narrative는 전이법의 변화다.
Narrative는 여러 경로의 파생 요약이다.
Narrative는 현재의 자기설명이다.
```

### 경계와 중복을 찾는다

현재 설명이 다른 흐름에서 이미 설명되는지, 하나의 이름 아래 서로 다른 종류의 것이 섞여 있는지 본다.

### 반례와 실패를 모은다

설명을 지지하는 사례만 쌓지 않는다.
이 설명이 필요 없거나 잘못되었다고 볼 수 있는 경우를 함께 둔다.

### 임시 발견을 남긴다

비교적 살아남은 결과는 기록하되, 영구 불변량으로 승격하지 않는다.
이후 흐름이 분리되면 발견도 이동할 수 있다.

---

## 12. 연구 문서의 자유와 최소 질문

현재 단계에서는 모든 문서에 고정 schema나 열 칸짜리 claim 형식을 강제하지 않는다.
형식이 생각보다 먼저 굳는 것을 피하기 위해서다.

다만 문서를 쓸 때 다음 질문을 참고한다.

```text
이 문서는 무엇을 붙잡으려 하는가?
현재 어떤 현상을 하나로 묶고 있는가?
왜 같은 문제 공간에서 다루는가?
다른 가능한 설명은 무엇인가?
어떤 개념이 섞여 있을 수 있는가?
어느 연구 흐름과 경계가 불분명한가?
어떤 관찰이나 모델 실패가 현재 생각을 바꿀 수 있는가?
이 문서가 맞다면 실제 수행에서 무엇이 달라져야 하는가?
이 문서가 틀리다면 어떤 모습이 나올 수 있는가?
```

문서의 역할에 따라 필요한 질문만 사용한다.
형식화가 필요한 주장이 실제로 생겼을 때 그 주장에 맞는 형식을 설계한다.

---

## 13. 모델 평가와 연구 흐름의 관계

모델 평가는 현재 연구 흐름을 검증하는 최종 재판이 아니다.
다음의 여러 원인이 섞여 있을 수 있다.

```text
연구 개념이 틀림
프롬프트 번역이 틀림
기반 LLM의 능력 한계
평가자가 현상을 잘못 읽음
대화 맥락이 충분하지 않음
현재 연구 흐름의 경계가 잘못됨
```

평가에서 문제가 나오면 어느 흐름으로 돌아갈지 바로 결정하지 않아도 된다.
경계가 불분명하면 `crossings/`에서 먼저 다룬다.

연구 흐름은 평가 결과를 받으며 다음처럼 변할 수 있다.

```text
질문이 더 구체화됨
두 설명이 실제로 구분됨
독립적으로 보였던 문제가 합쳐짐
하나로 보였던 문제가 갈라짐
현재 이름이 폐기됨
연구 자체가 중단됨
```

## 14. 현재 하지 않는 일

이 구조를 만든다고 다음을 주장하지 않는다.

- 다섯 연구 흐름이 인간의 다섯 불변 축이라는 주장
- 각 흐름이 독립된 인지 모듈이라는 주장
- Episode와 Narrative의 존재론이 확정되었다는 주장
- 권한 비도약이 인간 경험적으로 검증되었다는 주장
- 현재 분리가 최적이라는 주장
- 이후의 통합 이론 형태가 결정되었다는 주장

또한 현 단계에서는 여러 흐름을 조립한 상위 이론 디렉터리를 설계하지 않는다.
그 일은 각 흐름이 실제 연구를 통해 무엇으로 갈라지고 무엇을 공유하는지가 드러난 뒤의 과제다.

## 15. 구조가 잘 작동하는지 보는 기준

이 연구 구조는 다음 조건을 만족해야 한다.

### 새로운 질문을 기존 폴더에 억지로 넣지 않아도 된다

귀속이 불분명하면 `crossings/`에 둘 수 있어야 한다.

### 디렉터리 이름을 지키기 위해 개념을 왜곡하지 않는다

이름이 틀렸다면 바꾸고 이유를 남긴다.

### 흐름이 사라져도 연구 기억이 남는다

`transitions/`, `dormant/`, `discarded/`가 이를 담당한다.

### 같은 설명이 여러 곳에 복제되지 않는다

중첩은 지도와 링크로 표시하고, 아직 소유 위치를 정할 수 없다면 교차 문제로 둔다.

### 실제 연구가 구조를 바꾼다

구조가 연구를 고정하는 것이 아니라, 연구 결과가 구조를 다시 그려야 한다.
