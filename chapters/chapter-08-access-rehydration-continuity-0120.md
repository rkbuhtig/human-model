## 들어가며 — 과거가 남아 있다는 것과 지금의 내가 그것을 갖는 것은 다르다

Chapter 07의 인간은 부분 Coverage 위에서 가역 장면을 만들었다.

```text
Default Fog / Ψ
→ Reporter / Lens
→ Ego Compression
→ Cut-1
→ Cut-2
```

그러나 그 장면이 어디서 왔는지는 여전히 추상적이었다.

과거의 경험은 어디에 남아 있는가. 남아 있다면 왜 늘 떠오르지 않는가. 어떤 단서는 잊었다고 생각한 장면을 어떻게 다시 열며, 왜 장면 하나만이 아니라 감정·체감 부담·회피까지 함께 데려오는가. 그리고 그렇게 돌아온 것이 왜 곧바로 사실의 근거도, 지금의 ‘나’도 아닌가.

0120은 초기 HM·History·Ghost/Editor 계열에 이미 있던 접근 잠금·재구성·흉터 직관을, `W_t / F_t / 𝒢_t / A_t / Rehydration`의 일련의 문턱으로 다시 타이핑한다.

```text
축적 배경 F_t / 기억 그래프 𝒢_t
≠ 현재 Working Set W_t

주소 스캐폴드 ℐ
→ Access A_t
→ 국소 활성
→ Ψ 안의 Rehydration
→ 현재 장면과 정책의 변화
```

이로써 0120 모델 안의 과거는 현재를 직접 지배하는 유령 같은 장이 아니라, **접근될 때에만 현재 작동에 들어오는 잠재 구조**가 된다.

하지만 여기서 곧바로 자아 연속성이 생기지는 않는다.

```text
저장됨
≠ 주소를 가짐
≠ 접근됨
≠ 활성됨
≠ 재수화됨
≠ 선명함
≠ 사실의 근거
≠ 나의 경험으로 인수됨
≠ 현재 행동의 저자임
```

0120이 직접 만든 것은 대체로 앞쪽 문턱들이다. `self-adoption`, Episode provenance, Narrative identity, authorship handoff는 아직 없다.

따라서 이 장의 중심 명제는 이것이다.

> **0120은 과거에 현재로 돌아올 주소를 주었다.  
> 그러나 돌아온 과거에 자기 귀속·저자성·연속성을 부여하지는 않았다.**

그리고 이 장은 성공한 정의만 수거하지 않는다. 같은 날 안에 다음 과잉과 교정도 함께 남긴다.

- ClaimSig를 인증서처럼 오독할 위험과 ‘경로 제한자’ 교정
- Witness를 다시 Binding 성분으로 넣은 회귀
- `F_t`, `𝒢_t`, `Ψ_t`, `W_t` 사이의 타입 미봉합
- 모든 기억 접근을 유상 Forcing으로 만든 과잉
- Unknown을 근거 공백·비활성 기억·전략적 보류로 한 이름에 묶은 충돌
- 흉터가 회피를 필연한다는 식에서 OpenEase·AfterCost·재주소를 가르는 교정
- 의미를 좋은 변화, 모든 비영 간섭, 접근 기하 변화 사이에서 계속 바꾼 흔적

## 1. UC에서 UC+Q로 — 봉인된 표면에 합법적 미래 인과를 열다

### 1.1 0119가 남긴 문제

0119는 가역 후보와 근거 포트를 분리했지만, 표면·서사·정책이 현실에 어떤 양의 경로로 영향을 주는지는 `다음 policy` 수준에 머물렀다.

0120의 첫 문장은 인간을 다음 구조 위에 놓는다.

```text
닫힌 판정
+ 불변의 기준
+ 비가역 시간 화살
+ 표면이 미래 경로를 조향하는 합법 인터페이스
```

그리고 의미를 서사의 정태 내용이 아니라 `Φ → π → 미래 Π_wit/coverage → raw/σ 생성률`의 교차시간 효과로 옮긴다 `[MAIN20:L7–11]`.

이것은 중요한 방향 전환이다.

```text
표면이 현재 사실을 씀        금지
표면이 다음에 무엇을 만날지 조향  허용
```

### 1.2 S·Ψ·Φ·π의 동시 겹침

MAIN20은 인간을 순차 파이프보다 네 상태의 겹침으로 그린다.

```text
S  = 비가역 원장·귀속
Ψ  = 가역 후보장
Φ  = 표면·서사
π  = 정책·편성
```

네 상태는 각 tick에서 서로의 입력과 제약으로 얽혀 있다고 선언된다 `[MAIN20:L34–43]`.

이 그림의 장점은 “서사가 먼저, 행동이 다음, 현실이 마지막”이라는 단순 직렬 모델을 깨는 데 있다. 현재 장면·후보·정책·기존 원장은 동시에 서로를 제한한다.

그러나 `동시 겹침`이라는 말만으로 lawful edge가 생기지는 않는다. 무엇이 무엇을 어떤 지연과 타입으로 읽는지는 Port·Bandwidth·Cut으로 다시 적어야 한다.

## 2. Port와 Bandwidth — 내용보다 경로 자격을 먼저 묻다

### 2.1 세 포트

MAIN20은 0119의 input port 직관을 일반 인터페이스 법으로 확장한다.

| 포트 | 허용 입력 | 역할 |
|---|---|---|
| 근거 포트 | `Π_wit` 산출 | 판정 입력 |
| 정책 포트 | `Π_ctrl` 요약 knob | 다음 편성 |
| 표면 포트 | `Π_view` 산출 | 표현·장면·서사 |

`Bandwidth`는 각 포트가 허용하는 입력 정의역이고, 선언되지 않은 경로는 정의되지 않는다고 한다 `[MAIN20:L54–59]`.

여기서 얻은 규율은 강하다.

> **무엇이 강렬한가보다, 어떤 경로로 어떤 역할에 들어오는가를 먼저 검사한다.**

### 2.2 Authority Port라는 이름의 회귀

원문은 근거 포트를 괄호 안에서 `Authority Port`라고 부른다 `[MAIN20:L54–57]`.

그러나 Chapter 07에서 Witness는 권위를 가진 writer가 아니라 판정 입력 port로 교정되었다. 입력 자격과 상태 갱신 권한은 다르다.

현행 역할로 분해하면 `[CURRENT LENS]`:

```text
Evidence admission
≠ Warrant
≠ write capability
≠ Cut-2 application
```

따라서 이 장에서는 역사 인용을 제외하고 `근거 포트`라 부른다. `[REGRESSION]`은 기능보다 이름에 먼저 나타난다.

더 직접적인 회귀는 D15에서 일어난다. MAIN20은 Binding을 `B_ssot / B_wit / B_σ / B_other`로 다시 분해한다 `[MAIN20:L117–127]`. 그러나 0119 clean은 Witness가 Bind의 구성요소가 아니라 판정 입력 포트일 뿐이라고 명시했다 `[CLEAN19:L489–497]`.

```text
evidence admission
→ 다시 irreversible binding의 성분으로 혼입
```

이는 `[REGRESSION][REAL CONFLICT]`다.

### 2.3 선언된 경로와 합법 경로도 다르다

`Port/Bandwidth`를 선언했다고 그 경로가 정당해지는 것은 아니다.

```text
route declared
≠ route authorized
≠ payload valid
≠ evidence sufficient
≠ application warranted
```

MAIN20에는 누가 포트를 만들고 변경할 권한을 갖는지, 선언을 누가 인증하는지, payload 무결성을 어떻게 확인하는지가 없다. 이 공백은 ClaimSig와 Epoch에서도 계속된다.

## 3. ClaimSig — 서명처럼 보이지만 인증서가 아니다

### 3.1 MAIN20이 이 episode에서 먼저 고정한 tuple

MAIN20은 `모델 밖 항의 사건화`라는 제목 아래, 영향 주장에 경로 metadata를 붙이는 다음 tuple을 둔다. 다만 이 tuple 자체는 사건 기록이 아니다.

```text
ClaimSig := (Port, Bandwidth, Epoch, SourceTag)
```

- Port/Bandwidth: 어느 통로·정의역으로 영향이 주장되는가
- Epoch: 적용 규칙·메트릭·관할의 버전
- SourceTag: self/other/nature/unknown 같은 기원 표지

`[MAIN20:L151–157]`

이름 때문에 암호 서명, 권한 증명, 책임 서명으로 읽기 쉽다. 그러나 tuple에는 claim 내용, target, operation, evidence reference, issuer, jurisdiction grant, integrity proof가 없다.

### 3.2 PATCH20의 첫 교정: Signature-as-Constraint

PATCH20은 바로 이 오독을 봉인한다.

> ClaimSig는 진술·책임·원인 인증이 아니라 **귀속 이동이 허용되는 경계조건**이다.

불완전한 ClaimSig는 거짓이 아니라 `SSOT에 닿을 자격이 정의되지 않음`으로 처리된다 `[PATCH20:L14–19]`.

SourceTag도 blame·면책 토큰이 아니다. `unknown`은 누구의 책임도 아니라는 결론이 아니라 판정 유보와 추적 경로를 표시한다 `[PATCH20:L23–31]`.

```text
ClaimSig completeness
≠ truth
≠ evidence
≠ certification
≠ authority
```

이 교정은 살아남겨야 한다. 반대로 ClaimSig를 현행 typed envelope나 cryptographic signature와 동일시하면 안 된다.

## 4. Epoch — 불변을 값의 고정에서 변경 절차의 고정으로 바꾸다

MAIN20에서 Epoch는 규칙·메트릭·관할의 버전이었다.

PATCH20은 Epoch 변경 자체를 배경 변화가 아니라 `Commit-Cut`에 귀속되는 사건으로 만든다 `[PATCH20:L35–40]`.

```text
metric invariance
≠ metric can never change

metric change
= explicit event
+ version boundary
+ no silent retroactive rewrite
```

이것은 M2의 큰 진전이다. 불변을 “영원히 같은 값”이 아니라 **몰래 바꿀 수 없는 변경 절차**로 재해석한다.

하지만 아직 없다.

- Epoch change proposer
- change authority
- migration rule
- old/new comparison law
- rollback·revocation
- configuration provenance

따라서 Epoch는 현행 configuration version과 구조적으로 유사하지만 같은 타입은 아니다.

## 5. Hypothesis — 사실 권한 없이 행동을 조향하는 토큰

PATCH20은 `Hypothesis`를 `Π_view/Ψ` 내부의 행동 유도 가설로 정의한다.

```text
Hypothesis
→ π 조향 가능
↛ Π_wit 생성
```

`[PATCH20:L46–50]`

`[CURRENT LENS][BRIDGE]` 이 정의는 초기 Flux의 비근거 후보 자유와 Chapter 07의 Ψ를 행동 조향에 기능적으로 이어 읽을 다리를 만든다. 원문이 `Flux → Hypothesis`의 직접 승계를 선언한 것은 아니다.

> **가설이 사실이 아니어도 행동 후보를 만들 수 있다.  
> 행동을 만들었다고 가설이 사실이 되는 것은 아니다.**

PATCH20은 이어 view에서 시작한 행동도 사후 흔적·관측 변화·유보 기록 중 하나를 남겨야 한다고 주장한다 `[PATCH20:L55–63]`.

여기서 보존할 것은 `동기 ≠ 근거 ≠ 사후 귀속`의 분리다. 모든 행동이 반드시 debt나 명시 추적 의무를 남긴다는 보편식은 뒤에서 제한한다.

## 6. MeaningFlux — 의미를 내용에서 미래 차이로 옮기다

### 6.1 첫 정의군: 효과 흐름에서 좋은 방향의 간섭으로

MAIN20의 서두와 D17은 MeaningFlux를 Φ가 π를 거쳐 미래 근거·coverage·raw/σ 궤적에 남기는 효과 흐름으로 먼저 예고한다 `[MAIN20:L7–11, L135–138]`.

이어 TQ3는 이를 `Φ → π → 미래 Π_wit/coverage`의 교차시간 상관으로 닫고, 장기 `ΔQ⊥_raw` 또는 `σ` 생성률을 **낮추는 방향**만 의미로 남긴다 `[MAIN20:L320–327]`.

이 정의는 의미를 Repair와 결합한다.

### 6.2 두 번째 정의: 부호 없는 차이

PART M에서는 두 표면 조작이 미래의 raw·σ·coverage에 만든 차이를 비교하고, 차이가 0이 아니면 의미 간섭항이라고 정의한다 `[MAIN20:L458–467]`.

```text
MeaningInfluence
= future trajectory difference
```

여기에는 좋은 방향이라는 조건이 없다.

### 6.3 세 번째 정의: 생성률 미분

PATCH20의 첫 addendum는 MeaningFlux를 미래 raw/σ **생성률 변화**로 다시 고정한다 `[PATCH20:L78–81]`.

### 6.4 네 번째 해석: 접근 기하로의 가능한 구체화

마지막 vA0.7은 의미를 새 타입으로 확정하기보다, 어떤 단서가 어떤 기억 주변을 얼마나 쉽게 여는지의 장기 변화량으로 **해석할 수 있다**고 제안한다 `[PATCH20:L717–720]`.

따라서 MeaningFlux는 0120 안에서 하나의 안정된 정의가 아니다.

```text
유익한 생성률 감소
→ 미래 궤적의 비영 차이
→ 생성률 간섭 미분
→ 접근 기하 변화로의 가능한 구체화
```

이 이동 자체가 역사다. 최종 정본처럼 하나를 선택해 과거 문장에 덮어쓰지 않는다.

## 7. Working Set과 Field — 저장된 것과 현재 작동하는 것을 가르다

### 7.1 W_t: 유한한 현재

PATCH20의 두 번째 addendum는 tick `t`에서 실제 연산·의식·판단에 활성화된 유한 집합을 `W_t`라 부른다 `[PATCH20:L110–114]`.

```text
W_t = 지금 활성으로 올라온 제한된 조각
```

전체를 한 번에 펼쳐 최적화하는 주체는 정의역 밖으로 밀려난다 `[PATCH20:L135–138]`.

이것은 Reporter의 유한성을 기억·연산 쪽으로 옮긴다. 인간이 편향되기 때문에 제한적인 것이 아니라, 제한된 활성 위에서만 현재가 성립한다.

### 7.2 F_t: 축적 배경

같은 블록은 과거 커밋·경험·학습·사회 결합이 축적된 배경 상태를 `F_t`라 부른다.

`F_t`는 한꺼번에 전개되지 않고 Access를 통해서만 `W_t`에 영향을 준다 `[PATCH20:L115–124]`.

이때 초기 연구에 산재했던 구분이 UC+Q 문법 안에서 다음 비동일식으로 다시 명시된다.

```text
stored in F_t
≠ active in W_t
```

### 7.3 중요한 제한: F_t는 안정 승계되지 않는다

마지막 vA0.7은 `W_t`, `𝒢`, `ℐ`, Access를 재수록하지만 `F_t`를 직접 다시 정의하지 않는다.

```text
F_t = archive potential?
𝒢_t = archive structure?
F_t contains 𝒢_t?
𝒢_t represents F_t?
```

어느 관계도 닫히지 않는다. `F_t`는 중요한 발명인 동시에 마지막 재컴파일 밖에 남은 `[RESIDUE]`다.

## 8. Access — 조회를 사건으로 바꾸다

### 8.1 첫 접근 연산

PATCH20은 다음 함수를 둔다.

```text
A_t(F_t, π_t, context) → ΔW_t
```

Access는 대역 제한을 가진 `W_t` 갱신 사건이다 `[PATCH20:L121–125]`.

과거가 현재를 직접 갱신하지 못하고 Access를 거쳐야 한다는 봉인은 큰 수확이다.

```text
archive persistence
≠ current causal availability
```

### 8.2 Access-as-Forcing

같은 addendum는 Access가 단순 조회가 아니라 시스템을 구동하는 Forcing이며, coverage 변화·체감 부하·Receipt→σ 중 적어도 하나를 남긴다고 주장한다 `[PATCH20:L147–154]`.

마지막 vA0.7은 이를 더 강하게 재선언한다. `W_t` 밖의 항을 꺼내면 Receipt·coverage 공백·residual·PaidRepair 중 하나가 항상 남는다고 한다 `[PATCH20:L624–628]`.

여기서 두 명제를 분리해야 한다.

```text
Access may alter current state                   강한 수확
Every Access must hit one listed price category  추가 전역 가정
```

평범한 회상에도 계산·대사 비용은 있을 수 있다. 그러나:

```text
compute cost
≠ state backaction
≠ scar
≠ obligation
≠ Bill
```

이 가격표 논리를 모든 Access에 예외 없이 적용하면 Ghost의 완충지대까지 사실상 전부 유상 상태 전이로 재분류될 위험이 있다. `[CURRENT LENS]`

## 9. Persona — 자아가 아니라 접근 정책의 고정점

유한 Working Set 블록은 반복 상황에서 `π`가 Access가 무엇을 `W_t`로 올리는지에 대해 안정적인 조향 패턴을 만든다고 본다.

그 고정점·흡인자를 `Persona`라 부른다 `[PATCH20:L189–196]`.

```text
Persona
= recurrent access-selection pattern
```

이 정의는 페르소나를 캐릭터 설명문에서 실행 습관으로 옮긴다.

그러나 finite W와 반복만으로 수렴하는 고정점이 논리적으로 필연하지는 않는다. 학습 법칙·목적 함수·안정성 조건·다중 attractor가 정의되지 않았다.

그리고 더 중요하게:

```text
Persona fixed point
≠ current Ghost
≠ autobiographical identity
≠ diachronic self
```

Persona는 무엇이 쉽게 활성화되는지를 편향하는 정책 패턴이지, 활성 내용을 체험하고 행동을 인수하는 전체 자아가 아니다.

## 10. 기억 그래프 — 축적 배경에 국소 주소를 주다

### 10.1 𝒢_t

세 번째 addendum는 축적 상태를 그래프 `𝒢_t=(V_t,E_t)`로 바꾼다.

- node: 생각·경험 조각의 압축된 내부 packet
- edge: 연상·순서·공간·정서·행동의 연결 흔적

`[PATCH20:L228–235]`

그래프의 목적은 구현 자료구조를 강제하는 데 있지 않다고 문서가 말한다. 핵심은 **접근이 국소적이고 주변을 함께 데려온다**는 구조를 고정하는 것이다.

### 10.2 Archive writer의 부재

문서는 node가 저장된다고 말하지만 무엇이 그것을 쓰는지는 정의하지 않는다.

```text
experienced event
imagined candidate
heard story
later inference
public record-linked memory
```

이 다섯 종류가 같은 node가 될 수 있는지, provenance는 어디에 남는지, encoding 실패와 overwrite는 어떻게 생기는지 알 수 없다.

ClaimSig.SourceTag는 영향 주장의 경로 제한자이지 memory node provenance가 아니다.

따라서 `𝒢_t`는 archive read model의 시작이지 완성된 memory store가 아니다.

### 10.3 W_t의 타입이 바뀐다

첫 Working Set 정의에서는:

```text
W_t ⊂ Ψ_t
```

기억 그래프 블록에서는:

```text
A_t(ℐ, π, context) → W_t ⊂ V_t
```

`[PATCH20:L110–114, L246–250]`

`V_t`가 `Ψ_t`에 embed되는지, 기억 Working Set과 장면 Working Set이 같은 타입인지, 외부 Evidence가 어디에 활성화되는지는 없다. `[TYPE GAP]`은 이 장의 핵심 미완성이다.

## 11. 이미지 스캐폴드 — 내용이 아니라 주소

### 11.1 ℐ(v)

각 node에는 `Π_view`에서 렌더 가능한 감각·장면 조각의 껍질 `ℐ(v)`가 붙는다.

`ℐ(v)`는 Evidence가 아니라 그래프의 국소를 여는 주소 키다 `[PATCH20:L241–250]`.

마지막 vA0.7은 이를 한 문장으로 압축한다.

> **Π_view는 말할 권리가 아니라 찾아갈 권리를 제공한다.**

`[PATCH20:L619–633]`

이 문장은 0120의 가장 오래 남을 수확 중 하나다.

```text
vivid address
≠ strong evidence
```

### 11.2 image의 정의역 — 시각 전용인가 현상적 형상인가

초기 공리는 모든 graph node를 반드시 `Π_view`의 scaffold로 주소 지정하게 한다 `[PATCH20:L271–276]`. 앞의 직접 정의는 이를 단순 시각상이 아니라 `감각-장면 조각(이미지/감각 텍스처)`이라고 적는다 `[PATCH20:L241–245]`.

따라서 `image`를 시각 이미지 하나로 좁힌 이전 비판은 너무 강했다. 현재 Bridge에서는 두 경우를 나눠야 한다.

```text
ImageScaffold = visual imagery only
→ verbal / motor / affective / interoceptive path를 막는 과잉 폐쇄 위험

ImageScaffold = multimodal PhenomenalScaffold / QualiaMorph
→ 의식적 접근이 취하는 느껴지는 주소 형상의 강한 후보
```

다만 모든 graph 접근이 reportable `Π_view`를 반드시 거친다는 공리는 여전히 강한 아키텍처 선택이다. 절차 기억·습관·priming처럼 현재 의식 표상 없이 정책을 편향하는 경로가 있는지는 열어둬야 한다.

```text
ConsciousPhenomenalAccess may require QualiaMorph
≠ all implicit causal influence requires reportable qualia
```

보존할 핵심은 `주소 품질 ≠ 근거 품질`이다. 이 교정의 현행 상세는 Interchapter Note 08-A에서 별도 Bridge로 잠근다.

## 12. Rehydration — 압축 패킷이 현재 장면으로 펼쳐지다

Access된 node가 `W_t`에 올라온 뒤 Ψ 안에서 “아 맞다”의 구조로 펼쳐지는 과정을 `rehydrate(v)`라 부른다.

Rehydration은 내부 가역 Cut-1에서 일어나고 그 자체로 `Π_wit` 근거를 만들지 않는다 `[PATCH20:L252–255]`.

```text
compressed trace
→ local activation
→ reversible re-expansion
```

이 정의는 초기 HM 계열에 이미 있던 재구성 직관을 `압축 trace → activation → re-expansion`의 bounded-access 문법으로 다시 타이핑한다. 저장 내용을 그대로 재생한다는 모델과는 구분된다.

### 12.1 Recall의 세 문턱

PATCH20은 회상을 다음 합성으로 적는다.

```text
Render scaffold
→ Access local nodes
→ Rehydrate in Ψ
```

`[PATCH20:L299–307]`

이 사슬은 저장·접근·현재 체험을 분리한다는 점에서 강하다.

그러나 모든 회상이 이 순서로만 일어난다는 것은 `[MODEL ASSUMPTION]`이다. 자발 회상, 비시각 cue, 절차적 재현, body-first activation은 별도 경로를 요구할 수 있다.

### 12.2 Rehydration은 안정 승계된 최종 타입인가

마지막 vA0.7은 graph·scaffold·Access·backaction을 다시 정의하지만 `Rehydration`이라는 이름과 named operator는 재수록하지 않는다. 다만 Access 뒤 `W_t`에서 Ψ/Φ/π 후보 연산이 전개된다는 기능 흔적은 남는다 `[PATCH20:L648–651]`.

폐기라고 말할 근거도 없고 최종 안정 타입이라고 말할 근거도 부족하다.

```text
Rehydration = strong intermediate invention
             + final-block residue
```

## 13. 선명함과 근거 — 기억은 사실 인증서가 아니다

vA0.7은 scaffold의 선명함·감각성을 주소 품질로 한정한다.

선명함과 `Π_wit` 근거 강도는 단조 관계가 없고, 생생한 회상도 입력 자격을 스스로 만들 수 없다 `[PATCH20:L630–633]`.

또 회상 결과가 근거 lane에 들어오려면 외부 비가역 기록에서 재현 가능한 pointer를 동반해야 한다고 주장한다 `[PATCH20:L678–681]`.

여기서 두 층을 나눠야 한다.

```text
recollection vividness
≠ evidence strength

external pointer exists
≠ claim already true
```

외부 pointer가 주는 것은 검증 가능한 연결 가능성이지 판정 결과 자체가 아니다.

반대로 모든 autobiographical testimony를 외부 pointer 없이는 Evidence가 아니라고 닫으면, 오류 가능하지만 조사 가치가 있는 증언 lane까지 없앨 수 있다. 기억을 곧 사실로 만들지 않는 것과 기억을 어떤 증거도 될 수 없게 만드는 것은 다르다.

## 14. Backaction — 회상은 읽을 때 다시 쓴다

vA0.7은 Access가 graph weight나 scaffold 재구성에 역작용을 **남길 수 있다**고 정의한다 `[PATCH20:L658–661]`.

뒤 정리는 이를 더 강하게 “회상은 상태 갱신을 동반한다”고 적는다 `[PATCH20:L683–686]`.

```text
may leave backaction
≠ always creates material rewrite
```

두 문장의 강도는 같지 않다.

보존할 핵심은 다음이다.

> **기억 접근은 원본을 손대지 않는 파일 읽기와 같지 않을 수 있다.  
> 현재의 접근 경로가 다음 접근의 가중치를 바꿀 수 있다.**

하지만 backaction이 있다는 것만으로:

- 과거 사건이 바뀌는 것은 아니고
- 거짓 기억이 반드시 생기는 것도 아니며
- 현재 Ghost가 그 기억의 저자가 되는 것도 아니다.

```text
recollection modifies access state
≠ recollection authors past event
```

## 15. Unknown과 Forgetting — 서로 다른 부재를 한 이름에 묶다

### 15.1 첫 Unknown: 근거 공백

MAIN20의 Unknown은 `Coverage_view`가 `Coverage_wit`를 대신하지 못하는 데서 생긴다. 느끼고 상상하지만 근거로 확보하지 못한 영역이다 `[MAIN20:L71–75]`.

### 15.2 두 번째 Unknown: 비활성 기억

PATCH20은 graph에 구조가 있어도 현재 scaffold와 Access가 닿지 않아 `W_t`에 올리지 못한 영역을 Unknown이라 부른다 `[PATCH20:L337–341]`.

### 15.3 세 번째 Unknown: 전략적 보류

마지막 vA0.7은 고비용 이웃을 지금 열지 않는 것을 `주소 위생`이자 합법적 보류라고 부른다 `[PATCH20:L707–710]`.

세 Unknown은 같지 않다.

```text
Unknown_external
= 세계에 대해 근거가 없음

Inaccessible_archive
= 저장 구조가 있을 수 있으나 현재 접근 불가

Deferred_access
= 접근 가능성이 있어도 지금 열지 않기로 함
```

유한 Working Set만으로 `Coverage_view ⊄ Coverage_wit`가 논리적으로 필연한다는 정리는 과장이다. 유한한 시스템도 현재 view가 witness coverage 안에만 머무는 경우가 있을 수 있다.

### 15.4 Forgetting

PATCH20은 망각을 주로 scaffold 약화·접근 경로 변화·edge reweighting으로 설명하고 무흔적 삭제를 부정한다 `[PATCH20:L323–331]`.

이것은 **접근 실패형 망각**의 좋은 모델이다. 그러나 다음은 설명하지 않는다.

- encoding failure
- node deletion·decay
- storage damage
- source loss
- rehydration failure
- self-attribution loss

따라서 `망각 = 접근 재가중`은 한 유형이지 인간 망각 전체의 정의가 아니다.

## 17. LessonGradient — 흉터는 책임이 아니라 정책 압력이다

`LessonGradient`는 scar 주변 접근의 기대 비용이 정책 `π`에 대해 얼마나 민감한지를 나타내려 한다 `[PATCH20:L496–503]`.

원문이 붙든 핵심은 윤리 문장이 아니라 접근 비용이 다음 정책을 기울인다는 것이다.

그리고 바로 봉인한다.

> 비용·고통·교훈은 실재하지만 책임을 옮기는 권한 토큰은 아니다.

`[PATCH20:L552–555]`

```text
I was hurt
≠ therefore any attribution is true

I learned avoidance
≠ therefore I am blameworthy
```

다만 수학은 아직 열려 있다.

- π의 좌표와 미분 가능성
- 서로 다른 비용항의 단위·가중치
- `Δκ`를 비용으로 더할 수 있는 조건
- 기대값의 표본·시간 지평

이 없으므로 `LessonGradient`는 완성된 측정량보다 구조적 제안이다.

## 18. Propulsion과 Contraction — 강제 회피에서 재주소 압력으로

### 18.1 최초식의 과잉

첫 SCAR patch는 `LessonGradient ≠ 0`이면 정책이 접근 억제 방향으로 기울 수밖에 없다고 한다. 이후:

```text
대체 경로 개척 = Propulsion
coverage 축소·회피 고착 = Contraction
```

으로 나눈다 `[PATCH20:L409–424]`.

이 식은 흉터가 곧 회피를 결정한다고 너무 강하게 닫는다.

### 18.2 Refined의 기능 교정

Refined patch는 먼저 약한 `A-RESP` 공리를 추가한다.

반복 비용이 관측·누적되면 정책은 억제 **또는 재배치** 압력을 받지만 목적함수나 최적화 형태는 정하지 않는다 `[PATCH20:L509–515]`.

그 결과 최초의 `반드시 억제`가 다음으로 완화된다.

```text
scar + cost sensitivity
→ suppression or readdressing pressure
```

`[PATCH20:L521–527]`

이것은 같은 날 안의 실제 `[FUNCTIONAL CORRECTION]`이다.

### 18.3 여전히 남은 이분법

정제 후에도 결과를 Propulsion/Contraction 두 갈래로 닫는다.

그러나 가능한 반례가 있다.

- freeze
- oscillation
- repeated reenactment
- compartmentalization
- context-dependent split
- habituation·extinction
- 무반응·표류

또 안정적 Propulsion이 반드시 `Π_wit` 확장을 동반해야 한다는 문장은 성장의 한 규범적 정의이지 순수 정리가 아니다 `[PATCH20:L531–538]`.

안전한 휴식, 경계 설정, 신뢰 관계, 운동 학습, 받아들임처럼 witness coverage 확장으로 환원되지 않는 변화도 있을 수 있다.

## 20. 역사 본문의 결론 — 현재는 과거 전체가 아니라 접근 사건이다

0120의 하루는 다음 순서로 전개된다.

```text
UC+Q clean:
표면의 lawful future influence
→ Port / Bandwidth
→ ClaimSig / Epoch
→ Meaning as future interference

patch A:
서명·source·epoch·hypothesis 오독 봉인

≠ accumulated F_t
→ Access

patch C:
Memory Graph 𝒢
→ Scaffold ℐ

→ Rehydration

→ readdressing or contraction pressure

patch F:

```

이 지층은 인간을 전체 과거를 항상 펼쳐 가진 존재가 아니라, 제한된 현재에 일부 과거를 다시 불러 구성되는 존재로 **UC 형식 안에서 명시한다**.

하지만 그 인간은 아직:

- 무엇이 Archive에 기록되는지 결정하는 writer가 없고
- memory node의 provenance가 없고
- `F_t / 𝒢_t / Ψ_t / W_t` 타입이 하나로 봉합되지 않고
- 재수화된 기억을 자기 역사로 인수하지 않으며
- Episode 경계를 만들지 않고
- 몸과 행동 결과를 통해 저자성을 넘겨받지 않는다.

따라서 가장 정확한 역사 판정은 이것이다.

> **0120은 ‘과거가 나를 직접 지배한다’를  
> ‘과거는 현재 접근될 때에만 나의 작동에 들어온다’로 제한했다.  
> 그러나 접근된 과거가 왜 나의 것인지는 아직 설명하지 못했다.**

# 연구 후기 — 현재형 자아와 생명 모델로 역조명하기

## A1. 이번 장의 실제 수확

### A1.1 현재의 나는 Archive 전체가 아니다

0120이 연 가장 큰 관점은 `저장된 총합 = 현재의 나`를 깨뜨린 것이다.

```text
F_t
= 중간 patch의 accumulated background

𝒢_t
= 뒤 patch의 node-edge memory representation

relation(F_t, 𝒢_t) = undefined

Working Set
= 지금 실제로 작동하는 제한된 부분
```

현재적 자아가 있다면 그것은 Archive라는 물체 하나가 아니라, 매 순간 일부가 접근·활성·전개되는 사건적 실현에 가깝다.

> **과거가 남아 있어도 지금의 나에게 닿지 않을 수 있고,  
> 지금의 나에게 닿았어도 아직 나의 역사로 인수되지 않을 수 있다.**

### A1.2 View는 사실 발행권이 아니라 주소권이다

Chapter 07은 이미 `Π_view/StoryField`가 다음 `Π_ctrl`을 shaping하는 양의 기능을 허용했다. 0120은 그 기능을 처음 만든 것이 아니라, 기억 Graph를 여는 주소 scaffold와 future steering의 Port·Access 문법으로 확장한다.

```text
Π_view
↛ Evidence
→ address scaffold
→ Access
→ current candidate/experience
```

이것은 `non-grounding ≠ useless`를 기억 동역학 안에서 구체화한다.

### A1.3 잘 떠오름·아픔·진실·나임이 갈라진다

OpenEase와 AfterCost의 분리는 기억에 최소 네 축이 필요함을 드러낸다.

```text
Accessibility : 얼마나 쉽게 열리는가
Vividness     : 얼마나 생생하게 펼쳐지는가
AfterCost     : 열린 뒤 어떤 부담을 남기는가
Evidentiality : 외부 주장에 어떤 근거 자격이 있는가
```

현행 인간 모델에는 여기에 네 문턱을 더 분리해야 한다.

```text
PhenomenalMine-ness
= 지금 이 체험이 나에게 일어나는 것처럼 느껴지는가

SelfAttribution
= 이 기억·사건의 경험 주체를 나로 귀속하는가

NarrativeAdoption
= 그것을 현재 자기서사에 어떤 지위로 편입하는가

ActionAuthorship
= 실제 행동·결과의 행위자를 나로 판정하는가
```

0120은 Accessibility·Vividness·AfterCost·Evidentiality를 갈라놓기 시작했지만 뒤 네 문턱은 만들지 않는다.

### A1.4 흉터는 내용이 아니라 접근 기하를 바꾼다

흉터를 저장된 고통량 하나로 보면 왜 같은 단서가 어떤 날에는 열리고 다른 날에는 닫히는지 설명하기 어렵다.

0120의 refined scar는 더 좋은 질문을 연다.

```text
어떤 cue가 어떤 주변을 여는가
얼마나 쉽게 여는가
열린 뒤 무엇이 함께 오는가
그 경험이 다음 access policy를 어떻게 바꾸는가
```

이는 흉터를 고정 내용보다 **미래 접근의 지형 변화**로 읽을 길을 연다.

## A3. Ghost — 저장소가 아니라 현재 활성의 조립체

### A3.1 0120 부품으로 가능한 합성

최신 인간 모델의 렌즈로만 쓰면 다음과 같은 합성이 가능하다.

```text
[BRIDGE]
Ghost_t ≈
  CurrentInputTransform_t
  + WorkingSet_t
  + LocalCandidateField_t
  + CurrentQualiaMorph_t / FirstPersonSurface_t
  + SpontaneousResponseCandidate_t
  + AccessPolicy_t
  + StrategyPolicy_t

conditioned by
  Archive_t / Graph_t
  + BodyState_t
  + CurrentEnvironment_t
```

`SpontaneousResponseCandidate`는 raw input을 현재 상태에 맞게 변형했을 때 먼저 튀어나오는 가역 반응 후보다. Ghost의 핵심은 이 후보가 바로 외부화되지 않고 다른 후보·미래 trace와 함께 전략 선택을 거친다는 데 있다.

Body는 Ghost의 한 내용 성분이라기보다 무엇이 들어오고 무엇을 수행할 수 있는지를 조건 짓는 별도 actor·constraint다. 0120 원문에는 `Ghost`, CurrentInputTransform, SpontaneousResponseCandidate, BodyState를 묶은 이 식이 없다.

```text
narrow Ghost
= spontaneous candidate generator

broad Ghost-space
= generator + current experience
 + strategy / Editor mode가 함께 사는 현재형 자아 공간
```

Editor를 Ghost 내부 mode로 둘지 별도 sub-runtime으로 둘지는 여전히 `[OPEN]`이다.

### A3.2 Archive는 Ghost의 조건이지 Ghost 자체가 아니다

같은 Archive를 가진 상태에서도 context와 Access gate가 다르면 다른 현재가 실현될 수 있다.

```text
same stored graph
+ different cue
+ different access policy
+ different body state
→ different W_t
→ different current scene
→ different strategy
```

반대로 개별 episodic trace 일부가 손상되어도 몸·습관·관계·현재 목표와 남은 저장 구조가 연속성을 일부 보존할 수 있다.

```text
archive continuity
≠ sufficient for self continuity

complete retention of every trace
≠ necessary for self continuity

some cross-time retained substrate + handoff
= necessary candidate for diachronic selfness
```

저장 연결과 인수 handoff가 전부 끊기면 현재형 Ghost 반응은 다시 생길 수 있어도, 그것이 이전 Ghost와 같은 자아라는 통시적 근거는 사라진다. 그 경우 남는 것은 연속된 자아라기보다 transient Ghost realization이다.

### A3.3 Persona는 AccessPolicy의 attractor다

0120의 Persona는 반복적으로 어떤 조각이 쉽게 활성화되는지를 안정화하는 패턴이다.

따라서 현행 합성에서는:

```text
Persona_t ⊂ conditioning of Ghost_t
Persona_t ≠ Ghost_t
```

한 사람 안에 context별 여러 attractor가 있을 수도 있고, 하나의 attractor가 있어도 매번 같은 Ghost 장면이 생기지는 않을 수 있다.

## A4. 불연속적 자아 — 기억상실과 DID는 무엇을 시험하는가

### A4.1 직접 이론이 아니라 엣지 테스트

0120은 기억상실이나 해리성 정체감 장애의 임상 모델이 아니다. 진단·원인·치료를 이 장에서 주장하지 않는다.

다만 “전체 저장소가 남아 있어도 현재형 자아는 접근 구성에 따라 불연속적으로 실현될 수 있다”는 구조를 시험할 재료는 생겼다.

### A4.2 기억상실을 한 종류로 두면 안 된다

| 실패 위치 | 가능한 현상 | 0120 상태 |
|---|---|---|
| Encoding | 경험이 archive trace가 되지 않음 | writer 없음 |
| Storage | node·edge가 손실·손상됨 | 삭제 모델 없음 |
| Addressing | 적절한 cue/key가 생성되지 않음 | scaffold 약화로 일부 설명 |
| Access | trace가 W에 올라오지 않음 | 직접 모델링 |
| Rehydration | 활성됐지만 장면·구조로 펼쳐지지 않음 | 실패 타입 없음 |
| Source attribution | 내용은 있으나 출처·시점이 없음 | 타입 없음 |
| Self-attribution | 기억을 알지만 내 경험처럼 인수되지 않음 | 타입 없음 |
| Cross-state handoff | 상태 사이에 memory·authorship가 전달되지 않음 | 타입 없음 |

### A4.3 DID 엣지 모델

가능한 Bridge는 다음과 같다.

```text
shared body + partially shared Archive/Graph
        ↓
context-dependent AccessGate
        ↓
different WorkingSet partitions
        ↓
different AccessPolicy / Persona attractors
        ↓
alternating current Ghost assemblies
```

```text
context-dependent access partition
≠ distinct identity by itself
```

기분·기억·상태에 따라 다른 부분집합이 열린다는 사실만으로 별도 정체성이 성립하지 않는다. DID형 엣지 구조에 가까워지려면 반복되는 별도 access/persona attractor, self-model 차이, cross-state memory barrier, control/action handoff, authorship 배분, Episode·Narrative 귀속 차이가 지속적으로 결합되어야 한다.

사용자가 제안한 장기 스트레스 연결은 다음과 같은 제한적 `[BRIDGE]`로만 수거할 수 있다.

```text
repeated high AfterCost
+ context-coupled suppression / readdressing
+ scarred access weights
→ compartmentalized access basins
→ alternating Ghost assemblies
```

이는 “스트레스가 DID를 만든다”는 임상 정리가 아니라, Scar 반응이 Propulsion/Contraction 이분법 외에 `Compartmentalization` 분기를 요구할 수 있다는 구조적 반례다.

그러나 완성하려면 최소 다음 타입이 더 필요하다.

- ContextGate
- multiple AccessPolicy attractors
- ControlToken / action ownership
- CrossStateMemoryBarrier
- Episode provenance
- LivedByMe / SelfAdoption
- Ghost handoff
- shared-body authorship allocation

안전한 결론은 이 정도다.

> **현재적 자아는 전체 Archive가 항상 펼쳐져 있는 연속 물체가 아니라,  
> 제한된 접근과 Working Set으로 매번 실현되는 사건일 수 있다.**

이것은 장기 스트레스가 특정 장애를 만든다는 임상 인과 주장도, 인격의 모든 성질이 매 순간 완전히 단절된다는 주장도 아니다.

## A5. Editor와 자기 저자화 — 돌아온 과거를 누가 인수하는가

### A5.1 Reporter·Access·Editor의 차이

```text
Reporter
= 현재 관측 frame을 편성

AccessPolicy
= Archive에서 무엇을 현재로 활성화할지 편성

EditorFn
= 행동 branch, Episode boundary,
   Narrative adoption, authorship inheritance를 편성하고,
   미래 비가역 원장에 남을 trace를 예상해 선택
```

0120은 앞의 두 역할을 정교하게 만들지만 세 번째를 만들지 않는다.

`EditorFn ⊂ Ghost_t`라고 두더라도 Editor는 자유로운 주권자가 아니다. 이미 쓰인 Episode·Narrative·관계·몸의 비용이 만든 경사 위에서 제한된 후보를 읽고, 미래 기록을 예상하며 한 branch를 편성하는 유한한 기능이다.

```text
Ghost candidates
+ current Narrative terrain
+ projected irreversible traces
→ Editor deliberation
→ selected branch / hold
→ Actor–Body execution
→ World outcome
→ Episode boundary
→ later Narrative adoption
```

### A5.2 재수화된 기억의 가능한 운명

같은 RehydratedExperience에도 여러 응답이 가능하다.

```text
adopt as my lived past
hold as uncertain
attribute to another source
recognize as imagined
bind to an Episode but suspend meaning
accept occurrence but reject old interpretation
accept consequence but dispute authorship
```

이 분기 없이 `Rehydrated = my memory`라고 쓰면 접근 성공이 자기 귀속 권한을 자동 발행한다.

### A5.3 세계도 결과의 공동 저자다

Editor는 과거 결과를 마음대로 다시 쓰지 못한다.

```text
EditorFn : branch·boundary·adoption을 편성
Actor/Body: 실제 수행
World     : 결과의 공동 저자
Episode   : 발생·체험·흔적을 묶는 단위
```

회상 backaction은 현재 접근 상태를 바꿀 수 있지만 과거 사건 자체의 저자권을 넘겨주지 않는다.

## A6. 실제 경험과 서사 — 같은 사실이 왜 같은 자아를 만들지 않는가

사용자가 제시한 입양자 예시는 0120의 설명력과 한계를 동시에 보여준다.

```text
양육자가 나를 성인까지 길렀다.
그는 과거 흉악범이었다.
그 양육은 속죄의 일부였다.
```

외부 사실이 같아도 사람마다 다른 graph neighborhood가 열린다.

- 도구로 사용됐다는 배신
- 나를 살린 돌봄에 대한 감사
- 함께 보낸 실제 시간
- 현재 삶의 가치
- 범죄 피해자에 대한 책임
- 관계를 유지할 때의 위험과 비용

각자의 node·edge·순서·정서·body state가 다르므로 cue의 AddrSig, OpenEase, AfterCost, W_t, π도 달라진다.

Editor는 이 축들을 취향대로 설정하지 않는다. 실제 돌봄·배신·수리·공동 시간이 만든 접근성과 비용의 경사를 읽고, 그 경사를 따르거나 새로운 비용을 치르며 거스른다.

따라서 0120은 다음을 잘 설명한다.

> **판단은 추상 사실 목록만이 아니라, 실제로 살아온 연결 구조와 현재 접근되는 이웃 위에서 발생한다.**

하지만 어떤 해석을 자기 관계사로 인수할지, 양육자와의 Stake·Responsibility·Authorship·Identity·Authority를 어떻게 가를지는 설명하지 않는다.

```text
shared facts
≠ shared access geometry
≠ shared narrative adoption
≠ single correct personal response
```

이것은 사실 판정이 상대적이라는 말이 아니다. 같은 사실이 개인의 자기 경계와 미래 정책에 들어오는 경로가 다르다는 뜻이다.

## A7. 생명과 자기 경계 — 접근되는 내부가 곧 소유물은 아니다

### A7.1 0120이 다시 타이핑한 세 경계

현행 역할로 분해하면 0120은 적어도 세 경계를 다룬다.

```text
Phenomenal/operational boundary
= 지금 W_t에 활성화되어 사용 가능한 것

Epistemic-admission boundary
= 무엇이 Π_wit 근거 입력으로 들어갈 자격이 있는가

Authority/application boundary
= 무엇이 유효한 Cut-2를 거쳐 SSOT를 갱신할 수 있는가
```

0120은 첫째와 둘째를 강화했지만, `Authority Port` 명명과 `B_wit` 재삽입 때문에 둘째와 셋째를 완전히 안정 분리하지 못했다.

### A7.2 아직 없는 생명 경계

그러나 Archive와 Access만으로 생명이 되지는 않는다.

- 물질·에너지 유입
- 대사와 항상성
- body repair
- viability range
- 감각·행동 loop
- 번식·발달
- 몸–환경 경계

가 이 장에는 없다.

`F/𝒢`는 흐름 속에서 과거 패턴이 보존·재활성화되는 재료이지만, 생명이 그 패턴을 계속 유지하는 물리 과정은 아직 비어 있다.

> **[BRIDGE] 생명은 외부 흐름을 차단한 물체가 아니라, 흐름을 계속 통과시키면서도 에너지와 수리를 써서 특정 제약·패턴을 다시 결합하고 유지하는 과정일 수 있다. 정신적 자기 경계는 그 유지 함수 안에 어떤 사람·관계·상위 패턴의 존속과 손상을 포함할지 형성된 경계다.**

개체가 가족이나 집단을 위해 몸의 손실을 감수하는 현상은 유지 대상이 물리적 몸 바깥까지 확장될 수 있음을 시사한다. 그러나 그것만으로 집단이 하나의 생명체라는 사실이나 희생의 정당성이 자동 성립하지는 않는다.

### A7.3 내재화와 권한

타자에 대한 기억과 관계가 내 Graph에 깊이 들어와도 그 타자는 내 소유물이 되지 않는다.

```text
Belonging
Stake
Responsibility
Authorship
Identity
Authority
```

는 서로 환원되지 않는 readout이다.

이 축들은 현재 Ghost가 마음대로 돌리는 자유 knob가 아니다. 실제 Episode, 반복된 상호작용, 수리와 배신, Narrative adoption이 만든 관계 지형을 현재가 읽고 다시 인수하는 서로 다른 readout·commitment 축이다.

```text
Actual Episodes / care / betrayal / repair / promises
→ Narrative terrain
→ Belonging / Stake / Responsibility /
  Authorship / Identity / Authority readouts
```

가족의 존속과 손상이 내 유지 함수에 들어올 수 있지만, 그 가족을 대신 결정할 권한은 제한될 수 있다.

```text
Authority_felt
= 내가 대신 결정할 수 있다고 느끼는 내부 경계

ValidAuthority
= 관계·관할·동의·규칙이 실제로 부여한 통제 권한
```

둘도 같지 않다. 내재화가 `Authority_felt`를 키울 수는 있어도 `ValidAuthority`를 자체 발행하지 못한다.

대인 관계의 Authority와 원장 writer의 Authority도 구분해야 한다.

```text
LedgerAuthority
≠ InterpersonalAuthority

Responsibility_felt
≠ causal responsibility
≠ contractual/public obligation
```

0120의 `κ_other`는 공유된 비가역 결합 일부를 표현할 수 있으나 이 여섯 축 전체를 대신하지 못한다.

## A9. 형식 감사 — 0120이 아직 닫지 못한 것

### A9.1 W_t의 세 역할

```text

≠ consciously reportable working memory
```

### A9.4 Access의 가격표

모든 Access가 system state에 아주 작은 변화를 남길 수 있다는 주장과, 모든 Access가 `σ/Bill`을 낳는다는 주장은 다르다.

```text
AccessTrace
MetabolicCost
AttentionCost
LearningUpdate
ScarUpdate
NormativeDebt
```

를 분리해야 한다.

### A9.5 Unknown의 세 역할

근거 미확보, 비활성 memory, 전략적 보류는 다른 state다. 하나의 Unknown token으로 합치면:

- 세계에 정보가 없는지
- 내부에는 있지만 못 꺼내는지
- 꺼낼 수 있지만 안 여는지

구분할 수 없다.

### A9.6 Propulsion/Contraction 밖의 분기

흉터 반응은 적어도 다음 다중 상태를 허용해야 한다.

```text
approach
avoidance
readdressing
freeze
oscillation
compartmentalization
habituation
reenactment
context split
```

### A9.7 경로는 판정 내용을 대체하지 않는다

MAIN20의 “판정은 내용이 아니라 경로로만 성립한다”는 요약은 과장이다.

경로는 입력 자격을 정하지만 판정에는 payload·claim·rule·scope가 필요하다.

```text

≠ correct judgment
```

### A9.8 Qualia는 판정층에서만 gauge다

MAIN20은 서사·퀄리아를 chart-dependent gauge라고 부른다 `[MAIN20:L227–231, L490–494]`.

이것은 외부 판정가능항을 직접 바꾸지 못한다는 범위에서는 안전하다. 그러나 Qualia가 행동·기억 접근·자기 형성에 인과가 없다는 뜻으로 일반화하면 문서 자신의 `Φ→π`와 충돌한다.

```text
nonauthoritative
≠ noncausal
≠ unreal
```

### A10.2 ClaimSig에 없는 것

현행 typed record와 비교하면 ClaimSig에는 최소 다음이 없다.

- claim body / target
- operation type
- evidence reference
- issuer identity
- jurisdiction grant
- integrity proof
- certification state
- application receipt

따라서 `ClaimSig = Warrant` 또는 `ClaimSig = EventRecord`로 읽으면 안 된다.

### A10.3 Epoch에 없는 것

Epoch 변경을 사건화한 것은 선행 문제틀이지만, 현행형으로는 다음을 더 요구한다.
