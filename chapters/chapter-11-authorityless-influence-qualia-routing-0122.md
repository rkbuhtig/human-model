## 들어가며 — 느낌이 증거가 아니라면 아무것도 하지 못하는가

Chapter 09은 `ρ_fast`와 권위 원장 `𝔄` 사이의 빈자리를 발견했다 `[CH09:L1766–1779; L2033–2042]`.

```text
ρ_fast
: 순간 후보·상상·가역 편성

𝔄
: 사건·증거·귀속·책임의 원장
```

Chapter 10은 수면이 이 빈자리를 자동으로 채우지 않는다고 판정했다. 꿈·회상·재주소가 다음 상태에 영향을 줄 수 있어도, 그 영향이 무엇에 저장되고 얼마나 지속되며 다음 Ghost에 어떻게 넘어가는지는 정의되지 않았다 `[CH10:L1450–1482]`.

따라서 이번 장의 질문은 단순하다.

> `NEWQUAL22`의 `ΦΩ`는 그 물음표를 채우는가.

원문은 스스로 `ΦΩ`를 `qual/매질 메트릭`, `medium metric`, `impedance layer`라고 부른다. 가까움·막힘·거칠기·탄성을 표현하고, 후보가 call로 내려올 모델 확률을 기울이며, 장기 성형 주장에는 감사 가능한 흔적을 요구한다. Chapter 09의 현행 `QualiaMedium` Bridge는 별도 합성이다 `[CH09:L1170–1226]`.

이 이름만 보면 Chapter 09의 현행 Bridge인 `QualiaMedium`과 거의 같은 물체처럼 보인다. 그러나 직접 정의를 분해하면 다른 결과가 나온다.

```text
NEWQUAL22가 실제로 준 것
= 후보 분포를 기울이는 context-conditioned routing potential
 + 그 영향이 Evidence로 승격되지 못하게 하는 firewall
 + shaping claim의 audit metadata

NEWQUAL22가 주지 않은 것
= 현재 무엇이 느껴지는가
 + 경험이 무엇을 어떻게 바꾸는가
 + 그 변화가 어디에 얼마나 지속되는가
 + 다음 Ghost가 무엇을 인수하는가
```

이 장의 역사 판정은 다음 한 문장으로 미리 압축할 수 있다.

> **`NEWQUAL22`는 퀄리아를 가소적 매질로 만들지 않았다.  
> 퀄리아라는 이름 아래 비권위적 routing geometry를 만들고, 그 성형에 관한 주장을 감사 가능하게 만들려 했다.**

이 차이는 작지 않다.

```text
Metric / Potential
≠ Persistent Substrate
≠ Current Phenomenal Morph
≠ Plastic Trace
≠ Update Operator
≠ Audit Record
```

그러나 실패만 있었던 것도 아니다. `QUAL13`과 `UCXQ21`에 이미 있던 비권위적 영향 경로를, `NEWQUAL22`는 복원된 계보에서 처음으로 `P_call` 재가중 수식 수준에 형식화했다.

> **어떤 내부 상태는 사실을 증명할 권한 없이도  
> 다음에 무엇이 떠오르고 호출되고 행동 후보가 되는지를 실제로 바꿀 수 있다.**

이번 장은 이 `authorityless causal influence`를 살리고, 그것을 곧바로 퀄리아의 물질·기억·신경 가소성·자아 연속성으로 승격한 부분을 분리한다.

## 0. 범위·판본·독해 규율

### 0.1 주자료는 독립 이론이 아니라 삽입 패치다

`NEWQUAL22`의 정식 표제는 `Patch Addendum vA0.6-ΦΩU1 — ΦΩ-SEALING-UPGRADE (for v0.6-UCXQ)`다. 문서는 새 정본을 처음부터 쓰지 않고 세 삽입 위치를 지시한다.

1. UCXQ 최상위 봉인문장 아래 경고등 삽입
2. Annex `ΦΩ`의 목적문 아래 같은 경고등 재삽입
3. Annex 안에 정의·공리·정리 조각 추가

즉 현재 보존본은 완성된 퀄리아 이론 전체가 아니라 **기존 UCXQ에 붙일 안전 패치**다 `[NEWQUAL22:L3–11]`.

이 지위 때문에 다음을 구별한다.

```text
patch가 금지한 것
≠ patch가 설명한 것

patch가 참조한 Annex
≠ 현재 corpus에 완전한 독립 정본으로 보존된 Annex
```

### 0.2 직접 전사는 두 층이다

따라서 이 계보는 **동일 객체의 보존 계보**보다, `느낌은 권위가 없지만 미래를 바꾼다`는 문제의 기능 계보다.

### 0.3 사라진 `만족수면퀄리아통합`을 소급 정본으로 쓰지 않는다

## 1. `QUAL13` — 퀄리아를 안전한 표면으로 만든 첫 봉인

### 1.1 `Φ`는 상태가 아니라 결정론적 readout이었다

`QUAL13`의 목적은 퀄리아를 신비적 주권으로 두지 않는 것이었다. 문서는 `Φ`를 다음 요소가 합쳐진 표면으로 정의한다.

```text
Z_t : 현재 perc render
Q_t : 생각·이미지·의미 덩어리
g_t : 변화 방향
p_t : 당김 크기
Λ_t : 제약 압력의 성분
λ_t : 제약이 잘라낸 총량
ξ_t : 불일치
```

이 표면은 현재상의 촉감, 당김, 장력, 막힘을 보여 주지만 SSOT state가 아니며, 기존 신호에서 계산되는 결정론적 함수다 `[QUAL13:L23–36; L44–105; L109–124]`.

```text
Φ_t
= Readout_Φ(
    Z_t, Q_t, g_t, p_t, Λ_t, λ_t;
    background, budget, lock, mismatch
  )
```

이때 퀄리아는 내부에 무엇이 영구 저장되어 있는지를 말하지 않는다. 현재 상태에서 무엇이 어떻게 **보이는가**를 말한다.

### 1.2 풍부한 표면을 만들었지만 직접 쓰기 권한은 제거했다

`QUAL13`은 `Φ / Meta / Why-token`이 물리 포트·비용·안전 판정·commit에 직접 입력되는 모든 경로를 금지한다 `[QUAL13:L32–36]`.

후반 테스트는 더 강하다.

> `Φ/Meta on/off`가 물리·비용·렌즈 update·EventFire·Quench 결과에 영향을 주면 실패다 `[QUAL13:L391–405]`.

이 봉인은 퀄리아를 Evidence와 권위에서 분리하는 데 성공한다. 그러나 너무 강하게 읽으면 다른 문제가 생긴다.

```text
felt surface가 아무 결정에도 영향이 없다면
→ 왜 surface를 계산하는가
→ 어떻게 감정·당김·막힘이 다음 선택을 바꾸는가
```

원문은 완전한 무력화를 의도하지 않았다. `Φ`의 영향은 view 재표현을 거쳐 다음 tick 입력·후보로만 나타나며, 이성은 view에서 `Q_t`를 재배치해 다음 `Φ`를 유도할 수 있다 `[QUAL13:L117–125; L179–183]`.

따라서 이미 이 층에는 두 문장이 공존한다.

```text
Φ 자체는 writer가 아니다.
view 재표현·이성의 조작은 다음 tick 후보를 바꿀 수 있다.
```

필요했던 것은 이 둘 사이의 합법적인 인과 경로였다.

### 1.3 Meta와 Why-token은 체험 위로 떠오른 debug handle이다

`QUAL13`은 당김이 크고 제약에 많이 잘리며 진전이 멎을 때 `MetaFire`가 발생하도록 한다. 그 결과 생성되는 `Why-token`은 설명이나 사실이 아니라 실패 지점을 다시 다루기 위한 view-only signature다 `[QUAL13:L128–175]`.

```text
large pull
+ large cut
+ plateau
→ MetaFire
→ Why-token
→ re-expression / candidate reorder
```

이 구조는 현재 연구의 `LanguageAnchor`에 가까운 선행항이다. 그러나 `Why-token`은 언어 전체도, 기억 흔적도, 경험 substrate도 아니다.

### 1.4 `QUAL13`이 남긴 정확한 빈자리

`QUAL13`은 다음을 강하게 얻었다.

- felt intensity와 truth authority 분리
- 현재 readout과 SSOT state 분리
- 막힘을 디버그 가능한 handle로 변환
- same-tick self-promotion 차단

반면 다음은 비어 있다.

- 현재 readout을 만드는 지속 가능한 내부 상태
- experience/thought가 그 상태를 바꾸는 법
- 다음 tick 후보 재정렬을 수행하는 명시적 routing operator
- 변화의 persistence·decay·source provenance

`NEWQUAL22`는 후보의 정적 재가중 형식에는 부분적으로 답한다. 그러나 `QUAL13 Φ → E_{ΦΩ}` 변환과 next-tick update operator는 주지 않는다.

## 2. `UCXQ21` — 자유로운 Thought와 비싼 Judgment 사이

### 2.1 `ρ`와 `𝔄`가 분리되며 마음속 후보는 자유로워졌다

`UCXQ21`은 내부 가역 상태 `ρ_t`에 후보 `Ψ`, 장면·서사 `Φ`, 정책 편성, `Π_ctrl`의 원재료를 넣는다. `ρ` 안에서는 얼마든지 생각할 수 있지만, 그 자체는 근거가 아니다 `[UCXQ21:L48–55]`.

비가역 세계선 `𝔄`는 오직 Instrument 결과 `y`와 Cut-2 `Update`를 거쳐 바뀐다 `[UCXQ21:L85–137]`.

```text
ρ_t
→ free reversible composition
→ Cut-1 / Norm
→ call_t
→ Instrument
→ y_t
→ Cut-2 / Update
→ 𝔄_{t+1}
```

### 2.2 포트 분리는 influence와 warrant를 다른 관할에 놓았다

`UCXQ21`은 네 포트를 분리한다.

| 포트 | 역할 |
|---|---|
| `Π_view` | 이미지·퀄리아·서사·가설 |
| `Π_ctrl` | 정책이 읽는 요약 노브 |
| `Π_wit` | 근거·판정·귀속·commit 입력 |
| `Π_phys` | 기준·거리·제약·청구 법칙 |

`Π_view`에서 나온 것이 같은 tick에 `Π_wit` 형식으로 변환되는 함수는 정의되지 않는다 `[UCXQ21:L168–188; L536–552]`.

이것은 `느낌이 틀렸다`는 주장이 아니다.

```text
feeling may exist
feeling may guide inquiry
feeling may change a proposal

but

feeling cannot self-issue EvidenceLink
```

### 2.3 Thought는 후보를 만들고 Judgment는 call을 만든다

`UCXQ21`의 중요한 인간 모델은 Thought–Judgment split이다.

```text
Thought
= Π_view / Ψ 위의 연쇄 후보 생성

Judgment
= 실행·거부·보류를 결정하는 정책 연산

Gate
= Judgment가 Access를 여닫는 인터페이스
```

`Norm`은 이 가역 편성을 사건 호출 가능한 정규형으로 내린다. 유한 대역·지연·우선순위 때문에 모든 생각이 call이 되지는 않는다 `[UCXQ21:L102–126; L452–463]`.

여기서 `NEWQUAL22`가 들어갈 자리가 생긴다.

> 여러 후보 중 무엇이 더 가깝고, 더 부드럽고, 더 막혀 있고, 더 쉽게 call로 내려오는가.

### 2.4 생생함은 Evidence가 아니지만 call을 촉발할 수 있다

`UCXQ21`은 생생한 장면과 Evidence 강도 사이에 단조 관계가 없다고 못 박는다 `[UCXQ21:L973–993]`.

그러나 합법 경로로는 상상이 검증 호출을 만들 수 있다.

```text
imagination
→ verification call
→ Instrument
→ EvidenceLink or Unknown
```

상상이 근거인 것은 아니지만, **무엇을 검증할지 선택하게 만드는 원인**일 수 있다 `[UCXQ21:L1538–1547]`.

### 2.5 H≥1의 의미 이론이 인과층을 요구했다

`UCXQ21`은 현재 표면 차이의 operational meaning influence를 미래 `coverage / σ / ΔQ⊥` 궤적 차이로 정의한다 `[UCXQ21:L997–1023]`. 이것은 영향의 판정 기준을 주지만 그 인과 mechanism을 모두 주지는 않는다.

이 정의는 다음 압력을 만든다.

```text
표면 차이가 미래 trajectory 차이와 결박된다
→ 어떤 경로로 바꾸는가
→ 그 경로가 Evidence를 밀반입하지 않는가
→ 장기 변화는 어떻게 추적되는가
```

`NEWQUAL22`는 첫 질문에 routing 형식을, 둘째 질문에 firewall을 준다. 마지막 질문에는 persistence tracking이 아니라 shaping claim의 audit 조건만 덧붙인다.

## 3. `NEWQUAL22`의 출발점 — 두 종류의 월권을 막는다

### 3.1 첫 번째 위협은 느낌의 증거 승격이다

최상위 경고 `WΦΩ-1`은 다음을 한 묶음으로 놓는다.

```text
comfort
attraction
vividness
dread
certainty
```

이들은 routing/readout일 수 있지만 `Π_wit` 근거·안전·진리 토큰은 아니다. `(Π_view / ΦΩ) → Π_wit` 승격 규칙은 정의되지 않는다 `[NEWQUAL22:L21–26]`.

### 3.2 두 번째 위협은 사랑의 면책 토큰화다

`WΦΩ-2`는 `Love / Γ_void / setpoint`를 수렴 기준면으로만 두고, 옳음·안전·정당화·면책을 발행하지 못하게 한다 `[NEWQUAL22:L28–31]`.

만 직접 말한다.

### 3.3 세 번째 위협은 무기한 무상 성형이다

문서의 목적문은 `ΦΩ`가 call 분포를 통해 선택을 지배할 수 있다고 인정한다. 동시에 그 지배가 아무 audit linkage 없이 무기한 누적되었다고 주장하는 경로를 닫으려 한다 `[NEWQUAL22:L15–17]`.

따라서 문서의 전체 구조는 다음 두 질문으로 이루어진다.

이 두 질문은 분리해서 평가해야 한다.

## 5. `Medium Metric / Impedance` — 이름보다 좁은 실제 정의

### 5.1 직접 정의는 후보 위의 접근 비용 지형이다

`D-ΦΩ0`은 `ΦΩ`를 `Π_view / Π_rep` 후보장 위에 정의되는 `medium metric / impedance layer`라고 부른다. 그 기능은 가까움·막힘·거칠기·탄성을 통해 call 분포를 제약하는 것이다 `[NEWQUAL22:L50–53]`.

이 문장에는 네 역할이 겹쳐 있다.

| 이름 | 요구되는 것 |
|---|---|
| medium | 지속 가능한 상태·변형·전달 |
| metric | 후보 사이 거리와 metric law |
| impedance | 흐름에 대한 저항·주파수/경로 의존 가능성 |
| energy | 후보별 scalar potential |

뒤에서 실제로 정의되는 것은 마지막 하나에 가장 가깝다.

### 5.2 핵심 수식은 energy-reweighted candidate distribution이다

원문은 QG의 열림 커널과 `ΦΩ`의 에너지 지형을 둔다.

```text
K_open(v | ctx)
: candidate/open-neighborhood base kernel

E_{ΦΩ}(v | ctx)
: access energy / distance potential
```

그리고:

\[
P_{\mathrm{call}}(v\mid ctx)
\propto
K_{\mathrm{open}}(v\mid ctx)
\exp[-E_{\Phi\Omega}(v\mid ctx)]
\]

로 call 분포를 정의한다 `[NEWQUAL22:L69–84]`.

정규화항을 명시하면:

\[
P_{\mathrm{call}}(v\mid ctx)
=
\frac{
K_{\mathrm{open}}(v\mid ctx)
\exp[-E_{\Phi\Omega}(v\mid ctx)]
}{
Z(ctx)
}
\]

이며:

\[
Z(ctx)
=
\sum_{v\in V_{ctx}}
K_{\mathrm{open}}(v\mid ctx)
\exp[-E_{\Phi\Omega}(v\mid ctx)]
\]

이다. 연속 후보 공간에서는 합을 적분으로 바꾼다. 확률분포가 되려면 공통 measurable support와 \(0<Z(ctx)<\infty\)가 필요하지만 원문은 이를 명시하지 않는다.

이것은 `NEWQUAL22`의 가장 강한 구성적 기여다.

```text
candidate availability
× effective accessibility
→ call probability
```

### 5.3 “권위 없는 지배”가 처음 수식이 된다

원문은 `ΦΩ`의 dominance를 `P_call` 변화로만 정의한다. 어떤 후보를 더 쉽게 부르고, 더 오래 지연하고, 탐색폭을 줄이고, 우선순위를 바꾸는 것은 실제 인과다. 그러나 그것이 근거·안전·정당화가 되지는 않는다 `[NEWQUAL22:L141–145]`.

```text
causal influence
≠ evidentiary authority

selection dominance
≠ truth dominance
```

이것이 이번 장에서 가장 강하게 회수할 문장이다.

### 5.4 하지만 이것은 아직 metric도 medium도 아니다

`E(v|ctx)`는 후보 하나에 붙는 scalar potential이다. 수학적 metric이라면 보통 두 점 사이 거리 `d(u,v)`와 다음 조건이 필요하다.

```text
d(u,v) ≥ 0
d(u,v) = 0 iff u=v
d(u,v) = d(v,u)
d(u,w) ≤ d(u,v)+d(v,w)
```

원문은 이를 정의하지 않는다.

또 medium이라면 최소 다음이 필요하다.

```text
S_{t+1} = U(S_t, source, interaction, plasticity)
E_t = ReadRoutingField(S_t, ctx)
```

이 역시 없다.

따라서 역사적으로 안전한 타입은:

```text
E_{ΦΩ}
= context-conditioned RoutingPotential
```

이다.

### 6.3 그러나 핵심 수식에는 `K_ctrl`이 없다

같은 식이 필요하지만 없다.

더구나 `Π_rep`과 QG는 이 패치에서 사용되지만 `UCXQ21`의 기본 포트 목록에는 정의되지 않는다. 이들은 삽입 대상 통합본의 다른 지층을 가정한 것으로 보이지만, 현재 보존본만으로 완전한 정의역을 복원할 수 없다.

### 6.4 노브의 필드 목록만으로 의미 밀반입을 막을 수는 없다

`priority`, `seed_lock`, `allow_width`는 표면상 자원 노브지만 어떤 후보에 적용되는지에 따라 풍부한 내용을 인코딩할 수 있다.

현행에서 보존할 것은 `Π_ctrl`이 Evidence 필드를 직접 쓰지 못한다는 **권한 타입 봉인**이다. 노브가 의미를 전혀 운반하지 않는다고까지 말할 수는 없다.

## 7. `M_shape` — 매질 변화가 아니라 유효 routing shift

### 7.1 성형량은 연속한 call 분포의 발산이다

원문은 `E_{ΦΩ}` 자체의 좌표 변화가 아니라 call 분포의 변화를 성형량으로 삼는다.

\[
M_{\mathrm{shape}}(t)
:=
\mathcal D
\left(
P_{\mathrm{call}}(\cdot\mid ctx,t),
P_{\mathrm{call}}(\cdot\mid ctx,t-1)
\right)
\]

`𝔇`는 KL·JS·TV 같은 분포 거리/발산 중 하나를 고르고 문서 전체에서 고정한다 `[NEWQUAL22:L88–97]`.

이 선택의 장점은 분명하다.

> 내부 변수를 이름만 바꿔 놓고 “아무것도 달라지지 않았다”고 주장하는 것을 막고, 모델이 산출하는 effective call distribution의 변화에서 policy shift를 본다.

### 7.2 그러나 `M_shape`은 `ΦΩ` 고유 변화량이 아니다

`P_call`은 적어도 다음에 의존한다.

```text
K_open
E_{ΦΩ}
ctx
candidate support
control gates
normalization
```

따라서:

```text
E fixed
+ K_open changed
→ M_shape > 0

E changed
+ K_open changed in canceling direction
→ M_shape = 0
```

가 가능하다.

`M_shape`은 전체 effective routing의 변화는 감지하지만, 변화의 원인이 metric인지 context인지 base kernel인지 식별하지 못한다.

현행 이름은 다음이 더 정확하다.

```text
EffectiveRoutingShift_t
= D(P_call,t, P_call,t-1)
```

### 7.3 누적 발산은 지속 흔적이 아니라 경로 길이다

```text
P_0 → P_1 → P_0
```

로 돌아오면 최종 분포는 처음과 같지만:

\[
\mathcal D(P_1,P_0)+\mathcal D(P_0,P_1)>0
\]

일 수 있다.

따라서 `ΣM_shape`은 adjustment work나 trajectory variation을 나타낼 수 있지만 endpoint persistence를 증명하지 않는다.

```text
path length
≠ durable alteration
≠ PlasticTrace
```

### 7.4 gauge resistance의 범위도 행동 수준이다

`T-ΦΩE`는 같은 `P_call`을 만드는 좌표·표기 교체가 `M_shape`를 속일 수 없다고 말한다 `[NEWQUAL22:L209–213]`.

살릴 수 있는 것은:

```text
same effective call distribution
→ same behavioral routing readout
```

이다.

다음은 나오지 않는다.

```text
same P_call
→ same internal substrate
→ same experience
```

## 8. `ShapingSig`과 audit account — 변화의 증명과 변화 자체가 다시 섞이다

`D-ΦΩ5`가 허용하는 집계 원천은 다음과 같다.

\ge

를 놓고 `F`가 단조 비감소이며 `F(0)=0`이라고만 한다 `[NEWQUAL22:L148–159]`.

그러나:

\[

도 이 조건을 만족한다.

따라서 nonzero shaping이 positive audit cost를 요구하려면 최소:

\[

\]

와 단위·집계·causal linkage 조건이 더 필요하다.

### 8.5 “로그 없는 변화”는 존재론이 아니라 인증 규칙으로만 남겨야 한다

이 강화는 다음을 혼동한다.

```text
claim about change is auditable
change has an external receipt
change creates normative debt
```

사적 학습·습관화·생각·꿈·정서 변화는 공적 `Π_wit` 원장을 만들지 않고도 인과적일 수 있다.

> **감사 흔적 없이 내부 변화가 존재할 수 없다는 것이 아니라,  
> 감사 흔적 없이 특정 성형이 일어났다고 authority/audit-facing하게 인증할 수 없다.**

즉:

```text
No Silent Certified Shaping
```

이지:

```text
No Unlogged Internal Change
```

가 아니다.

## 9. delayed feedback와 Anti-Oracle — 자기 인증을 막되 정보까지 지우지 않기

### 9.1 Cut-2 결과는 같은 tick의 `ΦΩ` 입력이 아니다

`A-ΦΩ3a/3b`는 tick `t`에서 확정된 witness 변화와 `𝔄` 로그를 같은 tick의 `Norm / Π_ctrl / ΦΩ`가 관측하지 못하게 한다. 피드백은 최소 `t+H`, `H≥1` 이후에만 가능하다 `[NEWQUAL22:L162–177]`.

차단하려는 루프는 이것이다.

```text
commit C
→ immediately feel more certain about C
→ reinterpret that certainty as evidence for C
→ self-certify the same commit
```

이것은 설계 firewall로서 유용하다.

### 9.2 같은 tick의 모든 내부 변화가 금지된 것은 아니다

원문은 `Π_view` 내부의 가역 후보·주의 재배치를 같은 tick에도 허용한다 `[NEWQUAL22:L162–168]`.

따라서 정확한 봉인은:

```text
same-tick reversible reorientation
= allowed

same-tick witness result
→ self-justifying ΦΩ feedback
= undefined
```

이다.

### 9.3 `H≥1`은 장주기 자기확증까지 막지 않는다

다음 루프는 여전히 가능하다.

```text
ΦΩ_t biases call_t
→ call_t affects what gets measured
→ delayed result enters ΦΩ_{t+H}
→ later calls become more biased
```

따라서 delay만으로 calibration·독립 증거·exploration·source provenance를 대체할 수 없다.

### 9.4 Anti-Oracle의 안전한 잔여

`A-ΦΩ4`와 `T-ΦΩC`는 `ΦΩ` 출력만으로 미래 `𝔄 / Π_wit` 사실을 예언·복원하는 규칙이 없다고 한다 `[NEWQUAL22:L181–195]`.

정의역 봉인으로서 강한 문장은:

```text
felt certainty
cannot self-issue truth authority
```

이다.

그러나 내부 감각·불길함·숙련된 직감이 외부 사실과 통계적으로 상관될 가능성까지 금지하면 과하다.

```text
No Self-Warrant
≠ No Predictive Information
```

불길함은 숨은 단서를 요약한 signal일 수 있다. 그 signal은 조사 우선순위를 바꿀 수 있지만, 독립 검증 없이 사실 인증서가 되지는 않는다.

### 9.5 정리는 외부 발견보다 내부 규약의 귀결이다

`T-ΦΩC/D/E`는 대부분 앞선 공리와 정의를 다시 포장한다.

## E0. 감사 범위와 판정 규율

이번 외부 감사는 `ΦΩ`와 이름이 비슷한 이론을 찾아 정당화하는 작업이 아니다. 다음 네 질문만 분리해 확인한다.

1. 과학에 합의된 단일 qualia metric이 있는가.

4. 그 변화가 경험 자체의 저장이나 qualia-specific causality를 증명하는가.

| 등급 | 의미 |
|---|---|
| **강한 지지** | 여러 방법의 인과·관찰 결과가 제한된 명제를 지지 |

| **근거 부족** | 원문의 강한 동일시를 지지하지 않음 |

## E1. 합의된 단일 qualia scalar는 없다

### E1.1 IIT의 `Φ`는 이론 내부 대응량이다

IIT 4.0은 `Φ-structure`를 경험의 질과, 그 구조를 이루는 distinctions와 relations의 `φ` 값 합인 큰 `Φ`를 경험의 양과 대응시킨다. 그러나 이것은 **IIT가 제안하는 형식적 동일시**다. 사람의 임의 경험마다 qualia를 직접 읽어 낸 합의 측정 단위가 아니다. [Albantakis et al. (2023)](https://doi.org/10.1371/journal.pcbi.1011465)

따라서:

```text
IIT Φ
≠ NEWQUAL22 ΦΩ
≠ empirical validation of ΦΩ
```

다.

기호의 유사성은 계보도 증거도 아니다.

### E1.2 경쟁 이론의 핵심 예측도 아직 한 측정으로 수렴하지 않는다

2025년 Cogitate Consortium의 사전등록 적대적 협업은 초역치·고대비 시각 자극을 본 총 256명의 별도 세 집단에서 fMRI(`n=120`)·MEG(`n=102`)·iEEG(`n=34`)를 측정해 IIT와 GNWT의 예측을 대조했다. 일부 결과는 각 이론과 맞았지만, 양쪽의 주요 예측 일부가 도전받았고 이론의 계산적 핵심을 현재 방법으로 직접 시험하는 데 한계가 남았다. [Cogitate Consortium et al. (2025)](https://doi.org/10.1038/s41586-025-08888-1)

이 결과는:

```text
science has no settled model-independent qualia meter
```

라는 제한적 종합 판정을 지지한다.

그러나:

- IIT 전체가 반증되었다.
- GNWT 전체가 반증되었다.
- 현상 경험은 측정 불가능하다.

까지 말하지 않는다.

### E1.3 PCI는 상태 capacity proxy이지 경험 내용의 눈금이 아니다

PCI는 TMS로 뇌를 교란한 뒤 EEG 반응의 시공간 복잡도를 계산해 각성·수면·마취·일부 의식장애 상태를 해당 표본에서 구별했다. 그 연구가 제시한 것은 의식적 역학을 지탱할 가능성에 관한 유망한 **상태 수준 index**다. [Casali et al. (2013)](https://doi.org/10.1126/scitranslmed.3006294)

그러나 PCI는:

```text
how red this red feels
how painful this pain feels
which memory content is present
```

를 재는 척도가 아니다.

```text
ConsciousCapacityProxy
≠ PhenomenalContent
≠ QualiaIntensity
```

### E1.4 외부 판정

합의된 단일 qualia scalar가 없다는 것은 불가능성 정리가 아니다. 현재 측정은 서로 다른 대상을 잰다.

| 측정 | 직접 겨냥하는 것 | 자동으로 의미하지 않는 것 |
|---|---|---|
| accuracy | 1차 과제 수행 | felt vividness |
| visibility report | 보고된 가시성 | 순수한 경험 자체 |
| confidence | 판단 확신 | 경험의 세기·진실 |
| `meta-d′` | confidence의 오류 구별 효율 | phenomenology |
| PCI | 상태의 복잡·통합 반응 capacity | 특정 경험 내용 |
| decoding | 분류 가능한 신경 정보 | 1인칭 동일 경험 |

따라서 `ΦΩ`를 외부 과학의 established qualia metric으로 올릴 근거는 없다.


### E2.2 confidence에는 지각 외 변수가 섞인다

착시·사전확률·비대칭 보상을 조작한 2025년 연구에서는 비지각적 조작도 confidence를 바꿨지만 별도의 reproduction 척도에는 같은 변화가 나타나지 않았다. [Sánchez-Fuenzalida et al. (2025)](https://doi.org/10.1038/s44271-025-00257-y)

이는:

```text
Confidence ≠ Experience
```

라는 일대일 동일시의 기각을 지지한다. 다만 제한된 시지각 조작이며, 저자들도 controlled reproduction을 포함한 어떤 척도도 subjective experience만을 순수하게 측정한다고 보증하지 않는다. reproduction을 phenomenology의 ground truth로 승격하지 않는다.

그러나 confidence가 경험과 무관하다는 뜻은 아니다. 둘은 상관될 수 있지만 동일하지 않다.

### E2.3 적극적 보고는 지각 외 처리도 더한다

binocular rivalry에서 눈 움직임·동공으로 지각 전환을 추적한 연구는 적극 보고 조건에서 보이던 일부 전두 활성이 수동 조건에서 사라지고, 후두·두정 차등 활성은 남는다고 보고했다. [Frässle et al. (2014)](https://doi.org/10.1523/JNEUROSCI.4403-13.2014)

이 결과의 안전한 해석은:

> 일부 신경 활성은 경험보다 보고·내성·행동 요구에 관련될 수 있다.

이다.

다음은 나오지 않는다.

```text
no-report proxy perfectly reads experience
frontal cortex is irrelevant to consciousness
```

### E2.4 행동적 명령 반응 부재는 명령 수행 인지 부재가 아니다

6개 센터의 의식장애 성인 353명 연구에서 행동적으로 명령 반응이 없던 241명 가운데 60명, 약 25%가 fMRI나 EEG에서 명령에 따른 과제 수행 반응을 보였다. [Bodien et al. (2024)](https://doi.org/10.1056/NEJMoa2400645)

이 연구는:

```text
no observable behavioral command response
≠ no command-following cognition
```

을 강하게 보여 준다.

그러나 이는 6개 전문센터의 convenience sample 안 비율이지 일반 모집단 유병률이 아니다. 참가자의 약 35%만 두 modality를 모두 받았고 센터별 과제·분석도 이질적이었다. 양성 신경 반응이 한 사람의 전체 phenomenology를 읽어 낸 것도 아니며, 음성 반응이 의식 부재를 증명하지도 않는다.

### E2.5 NEWQUAL22에 대한 판정

`NEWQUAL22`가 comfort·vividness·certainty를 `routing/readout`으로 묶은 것은 권한 감사에는 유용하다. 그러나 이들을 하나의 `ΦΩ` scalar 또는 동일 readout family로 보는 외부 근거는 없다.

```text
felt vividness
confidence
metacognitive sensitivity
reportability
motor report
call probability
```

는 서로 얽힐 수 있지만 다른 타입이다.

## E3. 경험 뒤에는 변화가 남을 수 있다 — 그러나 경험의 복사본은 아니다

### E3.1 기억 인출은 언제나 read-only가 아니다

쥐 공포기억을 재활성화한 직후 편도체에 단백질 합성 억제제를 주면 이후 기억 표현이 손상됐지만, 재활성화가 없거나 6시간 뒤 투여하면 같은 효과가 없었다. [Nader, Schafe & LeDoux (2000)](https://doi.org/10.1038/35021052)

이는 특정 조건에서 재활성화된 기억이 다시 불안정해질 수 있음을 지지한다.

그러나:

```text
every retrieval rewrites every memory
memory was fully deleted
past phenomenal episode was erased
```

를 뜻하지 않는다.

### E3.2 인간 에피소드 기억도 reminder 뒤 새 정보와 섞일 수 있다

소규모 대학생 물체 목록 과제에서 첫 목록의 맥락 reminder 뒤 두 번째 목록을 배우면, 약 48시간 뒤 첫 목록 회상에 두 번째 목록 항목이 더 많이 침입했다. [Hupbach et al. (2007)](https://doi.org/10.1101/lm.365707)

이것은:

```text
retrieval / reminder
→ later source mixture
```

를 지지한다.

과거 경험의 원본이 그대로 저장되었다가 교체되었다는 증거는 아니다. reconsolidation 외에도 source confusion·interference 해석과 재현 경계가 있으므로 모든 회상이 원본을 덮어쓴다고 일반화하지 않는다.

### E3.3 engram 활성은 기억 관련 행동을 유발하지만 qualia를 직접 측정하지 않는다

공포학습 때 활성화된 생쥐 해마 치아이랑 뉴런을 표지한 뒤 다른 맥락에서 광활성화하면 freezing이 증가했다. [Liu et al. (2012)](https://doi.org/10.1038/nature11028)

이 결과가 직접 보이는 것은:

```text
tagged DG ensemble activation
→ memory-related freezing behavior
```

이다.

별도의 Ryan 연구에서는 단백질 합성 억제로 자연 회상이 어려워진 생쥐에서도 학습 당시 engram 세포의 연결성이 남았고, 직접 광활성화하면 기억 관련 행동이 유발됐다. [Ryan et al. (2015)](https://doi.org/10.1126/science.aaa5542)

Ryan 연구가 특정 과제에서 지지하는 제한된 분리는:

```text
impaired natural retrieval
≠ no retained engram-cell connectivity
≠ no optogenetic retrievability
```

그러나 freezing은 행동 지표다.

```text
engram activation
→ same past fear quale replayed
```

라는 결론은 나오지 않는다.

### E3.4 맥락 흔적과 valence 결합도 분리될 수 있다

생쥐에서 기존 해마 DG 맥락 engram을 반대 valence의 조건화와 함께 재활성화하자 회피·선호 반응 방향이 바뀌었다. 같은 전환은 BLA engram에서 동일하게 나타나지 않았다. [Redondo et al. (2014)](https://doi.org/10.1038/nature13725)

이 연구가 지지하는 안전한 타입 분리는:

```text
ContextTrace
≠ ValenceAssociation
```

이다.

이는 `QualiaMorph` 하나에 기억 내용·정서 방향·접근성을 모두 넣지 말아야 한다는 외부 근거가 된다.

### E3.5 외부 판정

넓은 수준에서 강하게 지지되는 것은:

> **학습·정서·재활성화와 관련된 사건 뒤 미래 수행·연합·접근성이 달라질 수 있다.**

이다.

다음은 과제·종 한정 지지다.

- 자연 접근 실패와 retained engram connectivity/optogenetic retrievability의 분리: 생쥐 공포기억 과제
- 재활성화 뒤 기억 갱신: 쥐 공포조건화와 소규모 인간 목록 과제
- 맥락 표현과 valence association의 부분 분리: 생쥐 공포·보상 과제

그러나 다음은 지지하지 않는다.

- 경험 자체가 하나의 물질로 저장된다.
- 과거 QualiaMorph가 그대로 복사된다.
- 모든 plastic change가 의식 경험에 고유하다.
- `P_call` 변화가 곧 engram·plasticity 변화다.

## E4. 정서·각성은 다음 접근 지형을 기울일 수 있다

### E4.1 정서 상태의 잔류 효과

사람에게 정서적으로 각성되는 자극을 먼저 보여 준 뒤 9–33분 후 중립 자극을 제시하면, 반대 순서나 중립 조건보다 지연 회상에서 중립 자극 기억이 향상됐다. 정서 인코딩 때의 일부 신경 패턴도 이후 중립 인코딩 중 재현됐다. [Tambini et al. (2017)](https://doi.org/10.1038/nn.4468)

이는:

```text
experience-correlated affective/neural state
→ later encoding bias
```

를 지지한다.

정서 qualia 자체가 매질에 그대로 저장됐다는 뜻은 아니다.

### E4.2 prediction error는 학습 신호이지 현상성의 단위가 아니다

원숭이 조건화에서 도파민 뉴런 반응은 학습 전 예상 밖 보상에서 학습 후 보상 예측 단서로 이동하고, 예상 보상이 생략되면 예정 시점에 감소했다. 이는 temporal-difference reward prediction error와 맞는다. [Schultz, Dayan & Montague (1997)](https://doi.org/10.1126/science.275.5306.1593)

그러나:

```text
dopamine prediction error
≠ pleasure quale
≠ surprise quale
≠ universal qualia update unit
```

이다.

### E4.3 불길함과 직감에 관한 설계 Bridge

`[BRIDGE / DESIGN INFERENCE]`

현재 출처 원장은 숙련 직감이나 불길함이 숨은 외부 사실을 얼마나 정확히 예측하는지 직접 시험하지 않았다. 따라서 여기서는 정보성을 경험 법칙으로 주장하지 않고, 다음 governance 경로만 제안한다.

```text
felt signal
→ hypothesis / priority / caution
→ independent check
→ evidence-backed claim
```

> **느낌은 탐색 우선순위를 바꾸는 routing input으로 사용할 수 있다.  
> 그 외부 정확성과 정보량은 별도로 검증해야 한다.**

## E5. 경험이 plasticity를 “쓴다”는 인과는 아직 분리되지 않는다

### E5.1 공통 원인 문제

다음 두 모델은 현재 많은 실험에서 같은 관측을 만들 수 있다.

```text
Model A
PhenomenalExperience
→ PlasticUpdate

Model B
UnderlyingProcessing
→ PhenomenalExperience
→ report

UnderlyingProcessing
→ PlasticUpdate
```

기억·학습 연구는 대개 행동·보고·신경·생리 변화로 plasticity를 추론한다. 현상적 질 자체가 독립적인 writer인지 분리하지 못한다.

따라서 현행 이론에서:

```text
experience-correlated process
→ possible plastic change
```

까지는 Bridge로 둘 수 있다.

```text
QualiaMorph itself writes the substrate
```

는 Open이다.

### E5.2 plasticity 전체를 보고 가능한 경험에 종속시킬 근거는 없다

앞의 기억·engram·정서 연구가 직접 측정한 것은 행동·연결·접근성·신경 상태의 변화이지, 각 변화에 필요한 conscious episode의 유무와 고유 기여가 아니다. 따라서 모든 plasticity가 보고 가능한 경험을 요구한다고 놓을 근거도 없고, 반대로 해당 변화가 전부 비의식적이었다고 단정할 근거도 없다.

그러므로 persistent state를 곧바로 `QualiaMedium`이라고 부르면 경험과 결박된 변화, 보고되지 않은 경험, 비의식적 학습·습관화·생리 조절을 한 타입에 잘못 포함하거나 배제할 수 있다.

현행에서 substrate-neutral 이름이 필요한 이유다.

```text
PlasticState
first

Qualia-specific Medium
only if independently justified
```

### E5.3 1인칭 사건은 지워 버릴 필요도 없다

외부 측정이 qualia를 직접 scalar로 주지 않는다고 해서 경험의 실재를 부정할 필요는 없다.

```text
PhenomenalEpisode
: first-person occurrence

ScientificProxy
: report / behavior / neural / physiological measurement
```

두 타입을 분리하면:

- proxy를 experience 자체로 과잉 동일시하지 않고,
- experience를 측정 불가능한 마법 토큰으로 만들지도 않으며,
- 여러 proxy의 수렴으로 제한적 추론을 할 수 있다.

## E7. 외부 감사의 종점

외부 현실은 `NEWQUAL22`의 가장 좋은 직관과 가장 강한 과잉을 동시에 분리한다.

살아남는 직관:

> 현재의 느낌·정서·접근 상태와 함께 일어난 처리는  
> 다음의 기억·주의·가치·행동 가능성을 실제로 기울일 수 있다.

내려야 하는 동일시:

> 그 기울임을 하나의 `ΦΩ` metric으로 직접 재거나,  
> 그 metric을 경험 자체·저장 매질·plastic trace와 동일시할 수 있다.

외부 과학에 가장 가까운 문장은 이것이다.

> **경험이 매질로 남는다고 말하기보다,  
> 경험과 함께 일어난 사건이 시스템을 바꾸고  
> 그 변화가 이후 경험과 행동의 접근 지형을 바꿀 수 있다고 말해야 한다.**

### A1.2 core에 남길 최소 문장

## A5. `P_call` 모델을 형식적으로 고치기

### A5.1 Model A — field-direct policy

후보 공간 `V_ctx`, base measure `μ`, support 조건을 두고:

\[
P_{\mathrm{call},t}(v\mid ctx)
=
\frac{
K_{\mathrm{open},t}(v\mid ctx)
\exp[-\beta_t E^{route}_t(v\mid ctx)]
G_t(v\mid ctx)
}{
\int_{V_{ctx}}
K_{\mathrm{open},t}(u\mid ctx)
\exp[-\beta_t E^{route}_t(u\mid ctx)]
G_t(u\mid ctx)\,d\mu(u)
}
\]

로 쓸 수 있다.

여기서:

```text
K_open
: candidate availability / generator prior

E^route
: affect/persona/access-dependent selection potential

G
: explicit gate/execution feasibility

β
: sensitivity / temperature
```

최소 조건은:

```text
K_open ≥ 0
G ≥ 0
βE is dimensionless
common measurable support is specified
0 < Z_t(ctx) < ∞
```

이다.

이 모델은 candidate-specific `E^route`가 call distribution을 직접 재가중한다. 따라서 strict knob-only는 아니다. 대신 `E^route`가 Evidence·Justification 필드를 쓰지 못하는 authority-type firewall을 유지한다.

### A5.2 Model B — strict knob-only policy

Knob-only path를 유지하려면:

```text
K_ctrl,t
= SummarizeControl(
    M^plast_t,
    E^route_t,
    ctx,
    budget
  )

AllowedKnobs_t
= ApplyKnobs(K_ctrl,t)
= (
    threshold_t,
    exploration_t,
    delay_t,
    width_t,
    priority_t,
    rate_limit_t
  )

P^knob_call,t
= Norm(
    K_open,t,
    AllowedKnobs_t,
    FeasibilityGate_t
  )
```

처럼 mediation을 명시해야 한다.

이 모델에서는 candidate-specific `E^route(v)`를 `P_call` 수식에 직접 넣지 않는다. `E^route`를 요약해 registry가 허용한 global/control knobs만 바꾼다. `FeasibilityGate_t`는 별도의 후보 실행 가능성 검사이며, affective salience나 candidate preference를 다시 주입하는 두 번째 route field가 아니다.

`Model A`와 `Model B`는 다른 policy class다. 하나로 뭉개지 않는다.

### A5.3 전체 분포 변화와 route-field 기여를 나눈다

아래 route-field counterfactual은 candidate-specific field를 갖는 `Model A`에 해당한다. `Model B`에서는 개별 field 기여 대신 각 allowed knob의 intervention effect를 따로 추정해야 한다.

관찰량:

\[
\Delta_{\mathrm{total}}
=
\mathcal D(P_t,P_{t-1})
\]

은 전체 routing shift다.

`E^route`의 기여를 보려면 `K_open`, context, support, gate를 가능한 한 고정한 counterfactual 비교가 필요하다.

\[
\Delta_{\mathrm{route}}
=
\mathcal D\left(
P(\cdot\mid E_t,K^\*,ctx^\*,G^\*),
P(\cdot\mid E_{t-1},K^\*,ctx^\*,G^\*)
\right)
\]

이 값도 internal substrate change를 직접 증명하지는 않는다. 다만 원인 귀속을 한 단계 개선한다.

### A5.4 divergence 선택도 타입화한다

```text
KL
: asymmetric; support mismatch에서 infinite 가능

JS
: symmetric divergence; sqrt(JS)가 metric

TV
: bounded metric on same measurable space
```

따라서 `KL/JS/TV 중 하나`를 임의로 고르는 대신:

- candidate support가 변하는가
- 방향성이 필요한가
- bounded scale이 필요한가
- sample estimation이 가능한가

에 따라 선택하고 version을 signature에 남긴다.

### A5.5 `M_shape`의 이름을 바꾼다

```text
M_shape
→ EffectiveRoutingShift
```

를 권한다.

`PlasticShaping`이라는 이름은 `M^plast`의 실제 before/after 변화와 source-bound update가 확인될 때만 사용한다.

별칭 숫자는 기본적으로 파일명·archive 위치를 위한 식별자다. `LIFE22C`와 `GAME22A/B`의 파일명은 0122지만 archive mtime은 01-23이며, `CORE26`·`SYNTH27`도 별칭만으로 정확 작성 시각을 보증하지 않는다. 내용 계보와 파일 시간 계보를 구별한다.

### A.2 provenance gap

`만족수면퀄리아통합`은 `SYNTH27:L1`과 `MINI27:L142, L241`에 입력·삽입 대상으로 언급되지만 현재 corpus에 독립 파일로 없다. 따라서 잃어버린 통합본의 내용은 직접 근거로 사용하지 않는다.
